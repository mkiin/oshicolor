# checkpoint

現在の作業状態をスナップショットとして保存する。

## 保存先

`{WorkspaceRoot}/.claude/sessions/` に以下の命名規則で保存:

```
YYYY-MM-DD-<topic>.tmp
```

`<topic>` は現在の作業内容を kebab-case で要約したもの。

## 保存内容

以下のテンプレートで記録する:

```markdown
# Checkpoint: <topic>

Date: YYYY-MM-DD HH:MM

## 完了した項目

- ...

## 現在の状態

- 作業中のファイル: ...
- 直近の変更: ...

## 判明した事実

- ...

## 未解決の問題

- ...

## 次にやること

- ...

## キーとなる判断・理由

- ...
```

## 実行手順

1. 現在の会話コンテキストから上記テンプレートの各項目を抽出する
2. `git diff --name-only` で変更ファイルを確認する
3. テンプレートを埋めてファイルに保存する
4. 保存したファイルパスをユーザーに報告する

## 注意事項

- 既存のセッションファイルがあれば追記ではなく新規作成する（古いものは残す）
- コードの詳細は書かない。ファイルパスと概要レベルで記録する
- 「次にやること」は具体的かつアクション可能な粒度で書く
