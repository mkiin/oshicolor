# load

`.claude/sessions/` に保存されたチェックポイントを読み込み、前回の作業状態を復元する。

## 実行手順

1. `{WorkspaceRoot}/.claude/sessions/*.tmp` を Glob で一覧取得する
2. 引数が指定されている場合はファイル名の部分一致で絞り込む
3. 引数が未指定の場合は最新（ファイル名の日付順）のチェックポイントを選択する
4. 該当ファイルを Read で読み込む
5. 現在の git 状態（`git status --short` と `git log --oneline -5`）を取得する
6. 以下のフォーマットでユーザーに報告する

## 報告フォーマット

```
## セッション復元: <topic>

**チェックポイント**: <ファイルパス>
**記録日時**: <Date>

### 前回の状態サマリ

- 完了: <完了した項目の要約>
- 未解決: <未解決の問題の要約>

### 次にやること

<「次にやること」セクションをそのまま表示>

### 現在の git 状態

<git status と直近コミットの情報>

### チェックポイントとの差分

<チェックポイント記録時と現在の状態に差異があれば報告>
```

## 複数候補がある場合

- セッションファイルの一覧を表示し、ユーザーに選択を促す

## 注意事項

- チェックポイントファイルが存在しない場合はその旨を報告して終了する
- チェックポイントの内容は要約せず、構造化して正確に伝える
- 「チェックポイントとの差分」では、記録時のコミットと現在の HEAD を比較する
