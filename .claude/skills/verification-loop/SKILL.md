# verification-loop

実装完了後に自動で検証ループを回す。

## 検証ステップ

以下の順序で実行し、各ステップでエラーがあれば報告する:

### 1. Lint チェック

```bash
vp lint
```

エラーがあれば:

- 自動修正可能なもの → 修正して再実行
- 手動対応が必要 → エラー内容を報告

### 2. ビルドチェック

```bash
vp build
```

エラーがあれば:

- 型エラー → エラー箇所と原因を報告
- import エラー → 修正を提案

### 3. テスト（テストファイルが存在する場合のみ）

```bash
vp test
```

失敗があれば:

- 失敗したテストケースと原因を報告

## 結果レポート

全ステップの結果をまとめて報告:

```
検証結果:
  lint:  PASS / FAIL (エラー数)
  build: PASS / FAIL (エラー数)
  test:  PASS / FAIL / SKIP (テストなし)
```

## 注意事項

- エラー修正は最小限にする。大きな変更が必要な場合はユーザーに報告のみ
- テストファイルがない場合は SKIP として報告（テスト不在の警告は出さない）
- 3回修正を試みても解決しない場合はループを止めてユーザーに報告する
