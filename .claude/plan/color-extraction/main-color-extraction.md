# メインカラー抽出アルゴリズム設計

> ステータス: 議論中
> 前提: キャラクター単体画像（背景除去済み）を入力とする

---

## 背景・動機

現行の node-vibrant / color-thief の Vibrant スロットは、キャラクターのメインカラーを
上手く拾えないケースがある。既存スロットのスコアリングは明度優先で占有率の重みが極端に小さいため、
面積が広くても目標値からわずかに外れた色が選ばれにくい。

**方針**: Vibrant スロットの選択アルゴリズムを本設計で置き換える。別フィールドは増やさない。

---

## 前提知識: アニメ/ゲームキャラクターの色設計理論

### 70:25:5 の面積比率ルール

| 役割 | 面積比 | 説明 |
|---|---|---|
| ベースカラー | 70% | キャラを象徴する色。衣装の大部分・髪色など |
| サブカラー | 25% | ベースを引き立てる支え役 |
| アクセントカラー | 5% | 目を引くポイント。瞳・リボン・武器の発光など |

→ **支持度（占有率 × 彩度）の上位クラスターがこの構造に対応する**

### キャラクターは基本的に2〜3色相で設計されている

MMCQ で得た代表色の色相分布には 2〜3 のピークが現れるはず。
それがデザイナーの設計意図を反映した色相。oshicolor はこのピークを正しく拾い、
画像にない第4・第5の色相を「発明」しない設計を目指す。

## アルゴリズム全体フロー

```
MMCQ出力（代表色N個）
  ↓
① 無彩色・極端な明暗を除外（フィルタリング）
  ↓
② 色相(H)を36°帯でクラスタリング（10クラスター）
  ↓
③ 各クラスターの支持度を計算
  ↓
④ 支持度順にソートし上位3クラスターを特定
  ↓
⑤ 整合性チェック（ベース/サブ/アクセントの妥当性検証）
  ↓
⑥ 各クラスター内からメインカラーを1色選ぶ
```

---

## ① フィルタリング

真っ黒・真っ白・無彩色を除外する。閾値は定数で定義し、実画像を通して調整する。

```
除外条件（OKLCH基準）:
  明度(L) < 0.08  → 真っ黒として除外
  明度(L) > 0.92  → 真っ白として除外
  彩度(C) < 0.05  → 無彩色として除外
```

**注意**: 背景除去が不完全な場合（薄い青空など）はフィルタリングで完全には対処できない。
ユーザーへの注意書きで対応する。

---

## ② 色相クラスタリング

OKLCH の色相(H)は知覚的均等空間上の角度のため、**36°固定の均等分割**で十分。

```
クラスター0:   0° 〜  36°  （赤系）
クラスター1:  36° 〜  72°  （橙系）
クラスター2:  72° 〜 108°  （黄系）
...
クラスター9: 324° 〜 360°  （赤紫系）
```

---

## ③ 支持度の計算

```
クラスターの支持度 = Σ（そのクラスターに属する色の 占有率 × 彩度）
```

**意図**: 面積が広く（占有率大）かつ鮮やかな（彩度大）色が多く属するクラスターほど支持度が高くなる。
70:25:5 理論のベースカラーは占有率が大きいため、自然に1位になりやすい。

---

## ④ 上位3クラスターの特定

支持度順にソートし、上位3クラスターを候補とする。

```
1位クラスター → ベースカラー候補
2位クラスター → サブカラー候補
3位クラスター → アクセントカラー候補
```

支持度が拮抗した場合（差が小さい）は支持度が高い方を素直に選ぶ。
フォールバック扱いとし、ユーザーが手直しする前提。

---

## ⑤ 整合性チェック

### ベースカラーとサブカラーの関係性チェック

キャラクターの配色理論から、ベースとサブは以下のどちらかに収束するはず。

```
類似色: 色相差 30°〜60°  → 統一感のある配色
補色:   色相差 150°〜180° → コントラストのある配色
```

これ以外の色相差（例: 90°前後）が出た場合、サブカラーではなくノイズの可能性がある。

### アクセントカラーの妥当性チェック

3位クラスターが本当にアクセントカラーかどうかを以下で検証する。

```
以下のいずれかを満たすこと:
  - 占有率がベースカラーより明確に小さい（5%前後）
  - ベースカラーとの色相差が大きい（補色寄り）
  - 彩度がベースカラーより高い
```

条件を満たさない場合はアクセントカラーではなくノイズとして除外する。

## ⑥ クラスター内からのメインカラー選択

各クラスター内で **population（MMCQ出力のピクセル数）が最大の色** を代表色とする。

```
理由:
- MMCQが出したpopulationはそのボックス内のピクセル数そのもの
- クラスター内で最も「密な」色として信頼できる
- 余計なスコアリングを重ねずMMCQの結果を尊重する
```
