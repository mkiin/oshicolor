# Qiita 記事の構成パターン

ryu-ki 氏の 85 本の記事分析から抽出された 8 パターン。
記事を書く前に「今回はどのパターンか」を決めると構成に迷わない。

ref: https://qiita.com/ryu-ki

## パターン早見表

| パターン           | いつ使う                        | 構成                                          | 目安文字数 |
| ------------------ | ------------------------------- | --------------------------------------------- | ---------: |
| 技術解説型         | ツール/サービスの機能を紹介     | はじめに → 概要 → 機能紹介 → まとめ           |      2,500 |
| ハンズオン型       | コードを動かす手順を記録        | はじめに → 準備 → 手順 → 実行結果 → まとめ    |      2,100 |
| 開発体験記型       | 何か作った過程と気づき          | はじめに → 作ったもの → 過程 → 感想 → まとめ  |      2,500 |
| イベントレポート型 | 勉強会/カンファレンスの記録     | 参加目的 → 概要 → セッション → 学び → まとめ  |      3,900 |
| トラブルシュート型 | エラー解決の記録                | 問題発生 → 原因調査 → 対処 → 確認 → まとめ    |      2,200 |
| 入門ガイド型       | 概念説明 + 実装を両方見せる     | はじめに → 概念 → 構成 → 実装 → 確認 → まとめ |      3,600 |
| 試してみた型       | 新しいものをサクッと触る        | はじめに → 触ってみる → 結果 → まとめ         |      1,400 |
| 概要紹介型         | 新機能/アップデートを画像で紹介 | はじめに → 機能紹介（画像多め） → まとめ      |      1,100 |

## 各パターンの詳細

### 技術解説型

最も汎用的。「〇〇とは何か」「〇〇でできること」を伝える記事。

```markdown
## はじめに

なぜこの技術/ツールを取り上げるのか

## 概要

何であるか、どういう位置づけか

## 機能/特徴の紹介

具体的な機能を見出しごとに紹介

## まとめ

要点の整理、所感
```

### ハンズオン型

読者が手を動かして再現できることがゴール。

```markdown
## はじめに

何をやるか、完成形のイメージ

## 前提/準備

必要な環境、バージョン、事前知識

## 環境構築

インストール、セットアップ手順

## 手順

ステップごとにコード + 説明

## 実行結果

動作確認、スクリーンショット

## まとめ

ハマりポイント、補足
```

### 開発体験記型

「作ったもの」より「作った過程で何を感じたか」が主役。

```markdown
## はじめに

動機、なぜ作ろうと思ったか

## 作ったもの

完成形の概要、スクリーンショット

## 開発の過程

技術選定、設計判断、つまずいた点

## 使ってみての感想/考察

良かった点、改善点、学び

## まとめ

今後の展望
```

### イベントレポート型

セッションごとに見出しを立てる。文字数が最も多くなりやすい。

```markdown
## 参加目的

なぜ参加したか

## イベント概要

名称、日時、テーマ

## セッション1: タイトル

内容の要約、印象に残った点

## セッション2: タイトル

（繰り返し）

## 学び/気づき

全体を通して得たもの

## まとめ
```

### トラブルシュート型

問題→解決の一本道。同じ問題にハマった人が検索で辿り着けるように。

```markdown
## 問題

エラーメッセージ、発生状況

## 原因調査

試したこと、調べたこと

## 対処/修正

解決策のコード/設定

## 解決確認

修正後の動作確認

## まとめ

原因の要約、予防策
```

### 入門ガイド型

概念と実装の両方をカバー。見出し数が最多で内容が厚い。

```markdown
## はじめに

対象読者、この記事で学べること

## 概念説明

背景知識、仕組み

## 構成の紹介

アーキテクチャ、ファイル構成

## 実装/コード

コード + 解説

## 動作確認

実行結果

## まとめ

次のステップ、参考リンク
```

### 試してみた型

最もコンパクト。速報性が重要。

```markdown
## はじめに

何が出たか、なぜ気になったか

## 触ってみる

実際に使った手順と感想

## 結果

どうだったか

## まとめ
```

### 概要紹介型

画像中心。スクリーンショットで見せる。

```markdown
## はじめに

何が新しくなったか

## 機能紹介

画像 + 短い説明の繰り返し

## まとめ
```

## 書き始める前のチェックリスト

1. **パターンを選ぶ**: 今回書きたい内容は上の8つのどれに近いか？
2. **構成を写す**: 選んだパターンの見出し構成をコピーして下書きにする
3. **見出しを埋める**: 各見出しに箇条書きでメモを入れる
4. **本文を書く**: メモを文章に展開する
5. **削る**: 冗長な部分を削る。目安文字数を参考に

## learn ディレクトリの記事との対応

このプロジェクトの `learn/` 配下の記事を書く場合:

- **トラブルシュート型**: `drizzle-kit-beta-esbuild-vulnerability.md`, `tsconfig-workers-types-warning.md`, `biome-a11y-interactive-div.md`
- **技術解説型**: `node-vibrant-vs-colorthief-swatches.md`, `typescript-declare-module-in-module-file.md`
- **試してみた型**: `typescript-array-flatmap.md`
- **ハンズオン型**: `vite-plus-github-actions-ci.md`, `colorthief-tutorial.md`
