# Product Overview: oshicolor

## Purpose

開発者がNeovimのカラースキームを自作する現在のワークフローは、大きく分断されている。まずCoolorsやAniColorsのようなパレット抽出ツールで画像から色を取り出し、次にlush.nvimやLuaの手書きでその色をNeovimのハイライトグループに割り当てる。この2段階のプロセスには1テーマあたり2-3時間を要し、色相・明度の調整やコントラスト比の検証が特に煩雑である。Neovimユーザーの多くはdotfiles管理やricing文化に積極的で、エディタの見た目へのこだわりが強い一方、テーマ作成のために色彩理論やアクセシビリティの知識まで持ち合わせているとは限らない。

同時に、アニメ・ゲーム文化と開発者コミュニティの交差点には明確な需要が存在する。Doki Theme（アニメキャラ別プリセットテーマ、19リポジトリ展開）の成功がそれを実証しているが、Doki ThemeはNeovim未対応かつプリセットのみで、ユーザーが自分の推しキャラからカスタムテーマを生成する手段を提供していない。AniColorsはアニメ画像に最適化されたパレット抽出を持つが、エディタテーマへの出力がない。つまり「画像入力→エディタテーマ出力」のワンストップソリューションは市場に存在しない。

本プロダクトは、キャラクターイラストや任意の画像をアップロードするだけで、コントラスト比やsyntax highlightingの識別性が自動調整されたNeovimカラースキーム（Lua形式）を生成するWebアプリケーションである。「推しキャラの色に囲まれてコーディングしたい」というEmotional Jobと、「最小限の手順で実用的なテーマを手に入れたい」というFunctional Jobの両方を同時に満たす。

## Target Users

### Primary Persona

20代後半の都市部在住Webエンジニア。Neovimを日常的に使用する中上級者で、dotfiles管理やricing文化に積極的に参加している。アニメ・ゲーム文化に親しみがあり、推しキャラクターのビジュアルから開発環境のカラースキームを自作したいという明確なニーズを持つ。現状はパレット抽出ツールとLuaの手書きによる2段階ワークフローを強いられており、この分断が最大のペインポイントとなっている。

### Use Cases

- Neovim中上級ユーザーが、新しいアニメのキャラクターイラストからコントラスト調整済みのLuaカラースキームを数分で生成し、dotfilesに組み込む
- Neovim初中級ユーザーが、Luaの知識なしにGUIプレビューで推しキャラのカラーテーマを作成し、ワンクリックでダウンロードする
- Linux ricing愛好家が、壁紙の画像からNeovimテーマを生成し、そのパレットをターミナルやWMテーマにも適用して環境全体の色を統一する
- ユーザーが自作テーマをコミュニティギャラリーに公開し、他のユーザーが閲覧・ダウンロードする
- ユーザーが生成したテーマのプレビュースクリーンショットをSNSやRedditに投稿してフィードバックを得る

## Key Features

- **ワンストップ画像→テーマ変換**: ユーザーが画像をアップロードするだけで、色抽出からNeovim Luaカラースキーム生成までを一貫して実行する。現状の「Coolorsで色抽出→手動でLuaファイルに組み上げ」という2段階ワークフローの分断を解消し、テーマ作成を数時間から数分に短縮する。これはプロダクトの最も核心的な価値であり、既存のどの競合も提供していない体験である

- **インテリジェントカラーマッピング**: 抽出したパレットの色を、Normal（背景・前景）、Comment、String、Function、Keyword、Type等のNeovim syntax highlightグループに自動でマッピングする。色相・明度・彩度の関係性を分析し、キャラクターの雰囲気を保ちながらコードの構造が視覚的に識別できる配色を生成する。Lua未経験のユーザーでもハイライトグループの知識なしにテーマを作れることが重要

- **コントラスト自動調整**: WCAG基準を参考に、背景色と各前景色のコントラスト比を自動検証・調整する。キャラクターの色彩をできる限り活かしつつ、長時間のコーディングに耐える視認性を確保する。画像から抽出した色がそのままsyntax highlightingに適するとは限らないため、この自動調整がテーマの実用性を左右する

- **リアルタイムプレビュー**: 生成されたテーマがNeovimでどのように見えるかを、実際のコードサンプル（TypeScript、Python、Lua等の複数言語対応）でブラウザ上にプレビュー表示する。ユーザーは個別の色をGUI上で微調整しながら結果を即座に確認できる。初中級者が「もう少し暗めに」「コメントをもっと控えめに」と直感的に操作できるUIが重要

- **ダーク/ライトバリアント生成**: 1つの画像入力から、ダークテーマとライトテーマの両バリアントを自動生成する。同じキャラクターの色彩を異なる明度環境に最適化して提供する。日中と夜間で異なるテーマを使い分けるNeovimユーザーの習慣に対応する

- **コミュニティギャラリー**: ユーザーが作成したテーマを公開し、他のユーザーが閲覧・ダウンロードできる共有機能。テーマにはソース画像のサムネイルとNeovimプレビューが付与され、キャラクター名やアニメ作品名でテーマを検索できる。catppuccin（7.2k Stars）が示すように高品質テーマへのコミュニティ需要は大きく、ネットワーク効果の源泉となる

- **パレットエクスポート**: 生成したカラーパレットをNeovim Lua形式に加え、hexコードリスト、CSS変数、JSON形式としても出力する。Linux ricing愛好家がターミナルエミュレータ（kitty、Alacritty）、ウィンドウマネージャ（Hyprland、waybar）、tmux等に同じパレットを適用して環境全体の色を統一するためのエコシステム拡張の基盤となる

## Market Context

AIカラーパレットジェネレーター市場はグローバルで約3.1億ドル規模（2024年）、年12%の成長が続いている。Coolors（月間200万ユーザー）やAniColorsのようなパレット生成ツールと、lush.nvim（GitHub 1.6k Stars）やcolorgen-nvimのようなNeovimテーマ生成ツールは、それぞれ独立した領域として存在しており、両者を繋ぐプロダクトは市場にない。Neovimは2025年のStack Overflow Developer Surveyで14%の採用率・83%の支持率を記録し、カスタマイズ文化が根付いたコミュニティを形成している。Doki Theme（アニメキャラ別プリセットテーマ）の成功はアニメ/ゲーム文化と開発環境パーソナライズの需要を実証しているが、Neovim未対応かつカスタム生成不可という制約がある。「画像から直接Neovimテーマを生成する」ワンストップ体験は明確な空白地帯である。

## Business Objectives

- リリース後6ヶ月で月間アクティブユーザー5,000人を達成する
- ユーザーによるテーマ生成数を月間10,000件以上にする
- コミュニティギャラリーへの公開テーマ数を1,000件以上蓄積する
- Neovim関連コミュニティ（GitHub、Reddit r/neovim、r/unixporn）での認知を獲得する

## Success Metrics

- **月間テーマ生成数**: 10,000件以上（プロダクトの核心的価値の利用頻度）
- **エクスポート完了率**: テーマ生成後に実際にファイルをダウンロードしたユーザーの割合70%以上
- **リピート利用率**: 月内に2回以上テーマを生成するユーザーが40%以上
- **ギャラリー投稿率**: テーマ生成ユーザーのうち10%以上がギャラリーに公開
- **GitHub Stars**: 関連リポジトリで初年度500 Stars以上を獲得

## Constraints & Assumptions

- 初期リリースはWebアプリケーション（SPA）として提供し、ネイティブアプリは開発しない
- 画像処理（色抽出）はクライアントサイドで実行し、ユーザーの画像をサーバーに送信しないプライバシー重視の設計とする
- MVP段階ではNeovim（Lua形式）のみをサポートし、Vim/VSCode/ターミナルエミュレータ等への対応は後続フェーズとする
- カラーマッピングロジックは初期段階ではルールベース（色相・明度による分類）で実装し、ユーザーフィードバックを基にした改善を段階的に行う
- 多言語対応（日本語・英語）を初期リリースから含める
