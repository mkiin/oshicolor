# oshicolor ロードマップ

## Pipeline v2（AI Vision ベース）

### ゴール

画像1枚から、キャラクターの色調に合ったテーマ（ダーク/ライト自動判定）の Neovim Lua ファイルをダウンロードできる。

### MVP 段階

| MVP | 概要 | 依存 | 状態 |
|---|---|---|---|
| MVP-1 | AI Vision で画像から色抽出+ラベリング → パレット JSON 出力 | なし | リサーチ中 |
| MVP-2 | パレット JSON → Neovim カラーマッピング → Lua ファイル生成 + ダウンロード | MVP-1 | 未着手 |
| MVP-3 | ブラウザ上でプレビュー表示 | MVP-2 | 未着手 |
| MVP-4 | ユーザー環境への配布方法 | MVP-3 | 未定 |

### MVP-1: AI 色抽出+パレット生成

画像 → Gemini Flash (Vision) → 色リスト+ラベル+象徴色 → パレット JSON

- AI Vision で色抽出+パーツラベリング+象徴色（primary/secondary/tertiary）判定
- キャラクターの色調からダーク/ライトテーマを自動判定
- AI 出力をパレット設計アルゴリズムに渡し、Neovim 用カラーパレット JSON を生成
- コントラスト保証（WCAG）、neutral 生成はアルゴリズム側で担保

### MVP-2: Lua 生成+ダウンロード

パレット JSON → Neovim ハイライトグループ割り当て → Lua ファイル出力

- pipeline-v1 の R4（Lua 生成）を流用・改修
- ダウンロードボタン UI

### MVP-3: プレビュー

- pipeline-v1 の R5（Neovim 風プレビュー）を流用・改修
- パレット変更時のリアルタイム反映

### MVP-4: 配布（未定）

- Neovim プラグイン連携
- GitHub Gist 出力
- その他

### MVP 後の方向性

MVP が動いたら、実際に自分で使ってみてフィードバックを得る。
改善は Issue ベースで優先順位をつけてから着手する。

#### 候補（優先順未定）

- 複数画像からのテーマ生成
