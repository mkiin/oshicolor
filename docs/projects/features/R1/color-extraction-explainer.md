# 色抽出の仕組み解説

`src/features/color-extractor/color-extractor.ts` が何をやっているかを、原理から説明するドキュメント。

---

## 全体の流れ

画像から「代表色」を選ぶ処理は、以下の 4 ステップで構成される。

```
画像（数百万ピクセル）
    ↓ ① サンプリング・量子化
「色グループ」への集約（数百〜数千グループ）
    ↓ ② フィルタリング
ノイズ・極端色を除いた候補 100 色
    ↓ ③ 貪欲法
視覚的に多様な N 色を選択
    ↓ ④ 色名付与
完成した ColorPoint[]
```

---

## ① サンプリングと量子化

### なぜ全ピクセルを処理しないのか

1000×1000 の画像には 100 万ピクセルある。全部処理すると時間がかかるうえ、隣り合うピクセルはほぼ同じ色なので処理する意味が薄い。そこで **8px おきに 1 点だけ拾う**（サンプリング）ことで、約 1/64 に削減している。

### 量子化とは

ピクセルの RGB 値は `(128, 63, 200)` のように連続した数値だが、そのままでは「似ているが完全に同じではない」色が大量に発生し、グルーピングができない。

**量子化**とは、RGB の各値を一定の幅（24 単位）で「丸める」処理のこと。

```
元の色:         R=128,  G=63,   B=200
↓ 24 単位に丸め
量子化後の色:   R=120,  G=72,   B=192
               (= round(128/24)*24)
```

これにより、近い色をひとつのグループにまとめられる。

```
ピクセル A: (127, 64, 199)  ┐
ピクセル B: (129, 61, 201)  ├─ 全部 "120,72,192" グループへ
ピクセル C: (126, 66, 200)  ┘

ピクセル D: (200, 10, 50)   ─── 別のグループ
```

### フィルタリング

以下の色はグループ化の前に除外する。

| 除外条件 | 理由 |
|---|---|
| 透明度 < 128 | 背景が透けている領域 |
| 明度 < 30 | ほぼ黒。エディタテーマで使いにくい |
| 明度 > 225 | ほぼ白。エディタテーマで使いにくい |

---

## ② 候補色の絞り込み

量子化後のグループを**出現回数の多い順**に並べ、上位 100 色を「候補」とする。

- 出現回数が 10 未満のグループはノイズとして除外
- 100 色に絞るのは、次の貪欲法の計算コストを抑えるため

この時点では**まだ最終的な色は決まっていない**。ただ「よく出てくる色の候補一覧」が手元にある状態。

---

## ③ 貪欲法による色の選択

ここが最も重要なステップ。「候補 100 色の中から N 色を選ぶ」という問題に対して、**貪欲法（Greedy Algorithm）**を使う。

### なぜ「出現回数トップ N 色」ではだめなのか

仮に出現回数の多い順に 5 色を選ぶとどうなるか。

```
キャラクター画像の例:
  1位: 薄いピンク（肌色）         ← 最頻出
  2位: やや濃いピンク（肌の影）   ← 2番目に多い
  3位: さらに濃いピンク（口）     ← 3番目
  4位: 白（目の白目）             ← 4番目
  5位: 薄橙（ハイライト）         ← 5番目

→ 5色が全部「似たような肌色系」になってしまう
```

oshicolor の目的はカラースキームの生成なので、**視覚的にバラエティのある色**が必要。

### 貪欲法の考え方

「すでに選んだ色たちから最も遠い色を次に選ぶ」を繰り返す。

```
候補の中から、すでに選ばれた色との「距離」が
最も大きいものを次に選ぶ
```

ステップを追って見てみる。

```
候補色（簡略化）:
  A: 薄ピンク（最頻出）
  B: 濃ピンク
  C: 紺色
  D: 黄色
  E: 黄緑

Step 1: まず A（最頻出）を選ぶ
  選択済み: [A]

Step 2: A から最も遠い色を選ぶ
  A→B の距離: 小さい（似ている）
  A→C の距離: 大きい ← 最大
  A→D の距離: 中くらい
  A→E の距離: 中くらい
  → C を選ぶ
  選択済み: [A, C]

Step 3: 「A との距離」「C との距離」のうち小さい方が最大の色を選ぶ
  B: min(A→B小, C→B大) = 小  ← B は A に近いので弱い
  D: min(A→D中, C→D大) = 中
  E: min(A→E中, C→E中) = 中
  → D か E を選ぶ（仮に D）
  選択済み: [A, C, D]

...以下繰り返し
```

この方法により、色が偏らずに「互いに離れた色」が選ばれる。

### 「距離」とは何か

OKLab という色空間での距離を使っている。

**RGB 距離の問題**:
RGB は数値としての距離と「人間の目で見たときの差」が一致しない。

```
例:
  青 (0, 0, 200) と 青 (0, 0, 100) の RGB 距離 = 100
  緑 (0, 100, 0) と 黒 (0, 0, 0) の RGB 距離   = 100
→ 数値は同じでも、人間の目では緑→黒の方が差が大きく見える
```

**OKLab の特徴**:
OKLab は「距離 1 = 人間が感じる色差 1」になるよう設計された色空間。距離が同じなら見た目の違いも同じになるため、「視覚的に多様な色を選ぶ」という目的に適している。

---

## ④ 色名の付与

選ばれた HEX 値（例: `#a3c4f2`）に対して、30,000 色以上の名前データベース（`color-name-list`）から**最も近い色名**を探す。

距離の計算はここでも OKLab を使う。

```
#a3c4f2 に最も近い色名は？
  → "Pale Cornflower Blue" (距離 0.03)
  → "Baby Blue" (距離 0.07)
  → "Periwinkle" (距離 0.09)
  → 最も近い "Pale Cornflower Blue" を採用
```

---

## 出力: ColorPoint

上記の処理を経て、最終的に以下の形式で N 個の色が返る。

```typescript
type ColorPoint = {
    id: number;     // 1 から始まる連番
    x: number;      // 画像上の位置（0〜1）※ 重心座標
    y: number;      // 画像上の位置（0〜1）※ 重心座標
    color: string;  // "#a3c4f2" 形式
    name?: string;  // "Pale Cornflower Blue"
};
```

`x`, `y` はその色グループのピクセルが画像のどのあたりに集まっていたかを示す（重心）。例えば髪の色なら画像上部、肌の色なら中央付近になる。

---

## 並び順のまとめ

| 位置 | 色の性質 |
|---|---|
| 1 番目 | 最も出現回数が多い色 |
| 2 番目以降 | それまでの選択色から最も「遠い」色（OKLab 距離） |

出現回数順でも色相順でもない。「できるだけ互いに似ていない色」を優先した順序になる。
