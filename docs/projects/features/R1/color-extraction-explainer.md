# 色抽出の仕組み解説

`src/features/color-extractor/color-extractor.ts` が何をやっているかを、原理から説明するドキュメント。

---

## 全体の流れ

```
画像（数百万ピクセル）
    ↓ ① ダウンサンプリング
約 40,000 ピクセルに縮小
    ↓ ② ピクセルフィルタ
透明・極暗色・極明色を除外
    ↓ ③ 2段階クラスタリング（extract-colors）
    |   RGB距離マージ → HSL距離マージ
色グループへの集約（数十〜数百グループ）
    ↓ ④ 結果フィルタ + ソート
lightness フィルタ → 面積降順
    ↓ ⑤ 上位 N 色を取得
    ↓ ⑥ 色名付与（OKLab）
完成した ColorPoint[]
```

---

## ① ダウンサンプリング

1920×1080 の画像には約 200 万ピクセルある。全部処理すると時間がかかり、隣り合うピクセルはほぼ同じ色なので処理しても意味が薄い。

`extract-colors` ライブラリに `pixels: 40_000` を渡すことで、ライブラリが内部で以下の計算を行いサンプリング間隔を決める。

```
reducer = floor(元画像ピクセル数 / pixels)
        = floor(2,073,600 / 40,000)
        = 51

→ 51 ピクセルごとに 1 点を読み込む
```

これにより常に約 40,000 点で処理が行われ、画像サイズに依らず処理速度が安定する。

---

## ② ピクセルフィルタ（colorValidator）

サンプリングされた各ピクセルは `colorValidator` で選別される。以下の条件を満たすピクセルのみクラスタリングに使われる。

| 条件        | 理由                               |
| ----------- | ---------------------------------- |
| alpha ≥ 128 | 半透明・透明領域を除外             |
| 輝度 ≥ 30   | ほぼ黒。エディタテーマで使いにくい |
| 輝度 ≤ 225  | ほぼ白。エディタテーマで使いにくい |

輝度は ITU-R BT.601 の知覚重みで計算する。

```
輝度 = 0.299 × R + 0.587 × G + 0.114 × B
```

---

## ③ 2段階クラスタリング

フィルタを通過したピクセルを「似た色のグループ」にまとめる処理。`extract-colors` ライブラリが2つの段階で行う。

### 第1段階：RGB 距離マージ（`distance` パラメータ）

出現頻度の高い色から順に処理し、`distance` 以内の色を吸収（マージ）する。

```
RGB距離 = (|ΔR| + |ΔG| + |ΔB|) / (3 × 255)
         = 0〜1 の範囲

distance = 0.08（デフォルト 0.22 より低め）
→ RGB の差が 8% 未満の色を同一グループとみなす
→ 低く設定することで類似色を分けたまま多くの色を残す
```

### 第2段階：HSL 距離マージ（`hueDistance` / `saturationDistance` / `lightnessDistance`）

第1段階の結果をさらに HSL 色空間でグループ化し、各グループの平均色を代表色として返す。

```
hueDistance        = 0.05  → 色相差 5% 未満を同系色とみなす
saturationDistance = 0.12  → 彩度差 12% 未満を同系色とみなす
lightnessDistance  = 0.12  → 明度差 12% 未満を同系色とみなす
```

全パラメータをデフォルトより低くすることで、類似色が早期にマージされることを防ぎ、12色以上の候補色が返るようにしている。

---

## ④ 結果フィルタ + ソート

ライブラリが返す `FinalColor[]` には各色の `lightness`（HSL、0〜1）と `area`（画像全体に占める面積比）が含まれる。

### lightness フィルタ

`colorValidator` をすり抜けた極端な明度の代表色を除外する。

```
0.08 ≤ lightness ≤ 0.92

→ #000000 は lightness = 0 なので除外
→ #ffffff は lightness = 1 なので除外
```

### area ソート

ライブラリのデフォルトソートは「鮮やかさ優先」だが、支配的な色（面積の大きい色）が先頭になるよう `area` 降順に並び替える。

```
area 降順:
  1位: 髪色（面積最大）
  2位: 肌色
  3位: 服の色
  ...
```

---

## ⑤ 上位 N 色を取得

フィルタ・ソート後の配列から `.slice(0, 12)` で上位 12 色を取り出す。

画像の色が少ない・フィルタで多く除外された場合は 12 色を下回ることがある。

---

## ⑥ 色名付与（OKLab）

選ばれた HEX 値（例: `#a3c4f2`）に対して、30,000 色以上の名前データベース（`color-name-list`）から最も近い色名を探す。

距離の計算には **OKLab** 色空間を使う。

### OKLab を使う理由

RGB は数値としての距離と「人間の目で見たときの差」が一致しない。

```
例:
  青 (0, 0, 200) と 青 (0, 0, 100) の RGB 距離 = 100
  緑 (0, 100, 0) と 黒 (0, 0, 0) の RGB 距離   = 100
→ 数値は同じでも、人間の目では緑→黒の方が差が大きく見える
```

OKLab は「距離 1 = 人間が感じる色差 1」になるよう設計された色空間。距離が同じなら見た目の違いも同じになるため、色名のマッチングに適している。

```
#a3c4f2 に最も近い色名は？
  → "Pale Cornflower Blue" (距離 0.03)  ← 採用
  → "Baby Blue"            (距離 0.07)
  → "Periwinkle"           (距離 0.09)
```

---

## 出力: ColorPoint

```typescript
type ColorPoint = {
    id: number; // 1 から始まる連番（area 降順）
    x: number; // 現在は 0 固定（extract-colors は位置情報を持たない）
    y: number; // 現在は 0 固定
    color: string; // "#a3c4f2" 形式
    name?: string; // "Pale Cornflower Blue"
    percent?: number; // 5.23（面積比を % 換算: area × 100）
};
```

---

## 並び順のまとめ

| 位置       | 色の性質                   |
| ---------- | -------------------------- |
| 1 番目     | 画像内で最も面積が大きい色 |
| 2 番目以降 | 面積降順                   |

出現頻度ではなく `area`（ライブラリが計算した面積比）の降順。面積の大きい色＝画像の印象を決める支配的な色が先頭に来る。
