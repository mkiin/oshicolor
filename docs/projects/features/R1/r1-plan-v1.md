## 参考実装との差分（改善点）

### 懸念点 1: RGB ユークリッド距離は知覚的に不正確

参考実装の貪欲法は `sqrt((r1-r2)² + (g1-g2)² + (b1-b2)²)` で距離を計算している。
RGB 空間は人間の知覚と一致しないため、アニメ画像では「RGB 上は離れているが視覚的には似て見える」色が選ばれる場合がある。

**改善**: OKLab 色空間での知覚的距離（`culori` の `differenceCiede2000` 等）を使用する。

### 懸念点 2: ライブラリの老朽化

| ライブラリ             | 状況                | 対応                              |
| ---------------------- | ------------------- | --------------------------------- |
| `nearest-color` v0.4.4 | ❌ 7年前に開発停止  | `culori` + 独自実装に置き換え     |
| `quantize` v1.0.2      | ⚠️ 開発停止の可能性 | MVP は不使用（貪欲法で代替）      |
| `react-color` v2.19.3  | ❌ 5年前に開発停止  | `react-colorful` に置き換え（R5） |
| `color` v5.0.0         | ✅ 継続中           | `culori` に統合できるため不要     |

### 懸念点 3: `culori` 1 本で R1・R3 を横断できる

- R1: RGB → HEX 変換、OKLab 距離計算
- R3: OKLch の L 値（明度）操作、コントラスト比計算

`color` ライブラリを `culori` に統一することで依存関係を削減できる。

### 懸念点 4: 座標系が 384 固定値でハードコード

参考実装は正規化座標を `0〜384` スケールで管理している（`Math.round(x / renderWidth) * 384`）。この定数は任意であり、oshicolor では **0〜1 正規化** に変更してよりクリーンな実装にする。

## 変更内容

### R1 アルゴリズムの記述

**変更前**: `k-means または中央値カット法（クライアントサイドTypeScript実装）`

**変更後**:

**主実装: 貪欲法（Greedy Algorithm）+ OKLab 知覚距離**

1. Canvas に画像を原寸描画し `getImageData()` で全ピクセル取得
2. 8px 間隔でサンプリング、透明（alpha < 128）・極端な明度（30未満/225超）を除外
3. RGB 各チャンネルを 24 単位で量子化してグルーピング、出現回数 ≥ 10 の上位 20 色を候補に
4. 貪欲法で多様性最大化：既選択色との **OKLab 知覚距離** が最大の色を順次選択
5. 各色に `culori` で HEX 変換し、`color-name-list` + 独自 nearest 検索で色名付与

### 実装定数（参考実装から確定）

| 定数               | 値                                          |
| ------------------ | ------------------------------------------- |
| サンプリング間隔   | 8px                                         |
| 量子化ステップ     | 24                                          |
| 透明度閾値         | alpha < 128                                 |
| 明度下限/上限      | 30 / 225                                    |
| 候補色フィルタ     | 出現回数 ≥ 10                               |
| 候補色上限         | 20 色                                       |
| 座標正規化スケール | **0〜1**（参考実装の 384 スケールから変更） |

### ColorPoint 型定義（参考実装から確定、座標スケールを修正）

```typescript
type ColorPoint = {
    id: number; // 1始まりの連番
    x: number; // 正規化座標（0〜1）← 参考実装の 384 スケールを変更
    y: number; // 正規化座標（0〜1）
    color: string; // HEX文字列 "#RRGGBB"（参考実装の rgb() 文字列から変更）
    name?: string; // 色名（color-name-list + nearest 検索で解決）
};
```

### 依存パッケージ（改善後）

**採用**:
| パッケージ | バージョン | 用途 |
|---|---|---|
| `culori` | ^4.0.2 | OKLab/OKLch 色操作・距離計算（R1 + R3 共通） |
| `color-name-list` | ^14.x | 色名データベース（活発にメンテナンス中） |

**不採用**（参考実装から除外する理由）:
| パッケージ | 理由 |
|---|---|
| `nearest-color` | 開発停止。`culori` で独自 nearest 実装に代替 |
| `color` | `culori` に機能統合されるため不要 |
| `quantize` | MVP は貪欲法で十分。開発停止の可能性あり |
| `react-color` | 開発停止。R5 で `react-colorful` を使用（別要件） |
