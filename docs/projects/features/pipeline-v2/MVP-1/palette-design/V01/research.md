# MVP-1/palette-design/V01 ghostty 463テーマ分析レポート

## 概要

ghostty に同梱されている 463 テーマの ANSI 16 色パレットを OKLCH に変換し、palette-design spec の定数を実績データで検証した。

分析スクリプト: `scripts/analyze-ghostty-themes.ts`

## G. spec 定数との比較

| 定数 | spec 値 | 実績 | 判定 | 備考 |
|---|---|---|---|---|
| `L_TARGET_DARK` | 0.75 | median=0.698, IQR=[0.613, 0.779] | ✓ IQR 内 | 中央値よりやや高め。0.72 に下げる余地あり |
| `L_TARGET_LIGHT` | 0.45 | median=0.585 | △ | 実績より暗め |
| `ERROR_CHROMA_MIN` | 0.12 | red C median=0.176 | △ | 実績より低い。0.15 への引き上げを検討 |
| `CHROMA_SCALE` | 0.9 | accent C median=0.132 | ✓ | AI 3色の C が 0.15 程度なら 0.135 で中央値一致 |
| `NEUTRAL_CHROMA_MAX` | 0.02 | bg C median=0.010, p75=0.024 | ✓ | 妥当 |
| `bg L range` | 0.10〜0.18 | median=0.230, p25=0.182 | ✗ | 上限が実績の p25。0.22 への緩和を検討 |
| `CONTRAST_AA` | 4.5 | bg vs accent p25=4.2 | ✓ | 実績より厳しいが正しい判断 |

## H. bat --theme=ansi の ANSI 色 → シンタックスマッピング

ghostty の `+list-themes` プレビューで使われる `bat --theme=ansi` の色割り当て:

| ANSI code | palette # | 色名 | シンタックス要素 |
|---|---|---|---|
| `[35m` | 5 | magenta | keyword 全般 (import, const, if, return, class, export, 演算子) |
| `[32m` | 2 | green | string, template literal, comment |
| `[33m` | 3 | yellow | number, boolean, type 名, builtin (console) |
| `[36m` | 6 | cyan | parameter, method call |
| `[34m` | 4 | blue | function 定義名 |
| `[0m` | — | fg | 変数名、括弧、その他 |

**所見:** 6 色中 5 色 (palette 2〜6) のみ使用。palette 1 (red) はシンタックスには未使用（エラー表示専用）。

## I. oshicolor への示唆

### 定数の調整提案

| 定数 | 現 spec | 提案 | 根拠 |
|---|---|---|---|
| `L_TARGET_DARK` | 0.75 | **0.72** | 実績中央値 0.698 に近づける |
| `ERROR_CHROMA_MIN` | 0.12 | **0.15** | palette 1 (red) 実績中央値 0.176 |
| `bg L range` | 0.10〜0.18 | **0.10〜0.22** | 実績 p25=0.182, median=0.230 |

### neutral 彩度の色相依存性

実績データから、bg の彩度は色相によって大きく異なる:

| キャラの色相帯 | bg C の実績 | 推奨 |
|---|---|---|
| 紫〜青紫 (240-300) | 0.019〜0.023 | spec の 0.015 で安全 |
| 青 (210-240) | 0.022 | 0.015 で安全 |
| 赤〜橙 (0-60) | 0.017〜0.043 | 0.015 で安全だが少数派の領域 |
| 黄〜緑 (60-150) | 0.009〜0.019 | 0.010 以下に抑えないと「汚い」リスク |

→ MVP-1 では一律 0.015 で実装し、24 キャラ SVG 検証で黄〜緑系キャラの見え方を重点確認する。
