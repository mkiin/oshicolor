# R2/V7 seed 数固定化（V + DV/LV 競合選定）

## なぜ V7 が必要か

V6 では各軸に V/DV/LV の 3 target を適用し、色がある限り取得する方式を採用した。
しかし軸内の色数に応じて seed 数が 1〜3 でばらつき、後段（tonal palette → ロール割り当て）の設計が不安定になる問題が判明した。

特に accent 軸は色数が少なく、DV は 67%、LV は 41% でしか取得できない（genshin 98キャラ中）。

一方、LV がキャラクターらしさを表現するケースがあるため、LV を単純に廃止するのではなく、DV と LV を競合させて良い方を採用する方式に変更する。

## 前版との変更対照表

| 項目 | V6 | V7 |
| ---- | ---- | ------ |
| 2色目の選定 | DV/LV を独立に取得（最大3色） | DV と LV の距離を比較し近い方を採用（固定2色） |
| seed 数 | 可変（軸あたり 1〜3） | 固定（軸あたり 2） |
| 合計 seed 数 | 可変（5〜9） | 固定（6） |
| フォールバック | 閾値段階緩和 | DV/LV 競合（閾値不要） |
| accent の seed 数 | 可変（1〜3） | 固定（2） |

## 設計方針

### 核心: V + DV/LV 競合選定

各軸で 2 seed を選定する:

```
1色目: V (Vibrant) target で最も近い色を選定（必ず取れる）
2色目: V で選ばれた色を除外し、残りの色について
        DV target への最小距離 d_DV と
        LV target への最小距離 d_LV を比較
        → d_DV < d_LV なら DV を採用
        → d_LV <= d_DV なら LV を採用
```

これにより:
- seed 数が常に 6（3軸 × 2色）で安定する
- キャラによって DV/LV の適切な方が自動選択される
- 閾値パラメータが不要になる（常に最小距離の色を取るだけ）

### seed 構成（固定 6 seeds）

| seed | 軸 | 1色目 | 2色目 |
| --- | --- | --- | --- |
| main-V | main | V target | — |
| main-DV or main-LV | main | — | DV/LV 競合 |
| sub-V | sub | V target | — |
| sub-DV or sub-LV | sub | — | DV/LV 競合 |
| accent-V | accent | V target | — |
| accent-DV or accent-LV | accent | — | DV/LV 競合 |

### 軸内の色が 1色しかない場合

2色目は V と同じ色が選ばれる（重複）。tonal palette 生成時に同一 seed は1つにまとめる。

## 変更内容

### 実装タスク

1. **selectSeeds の書き換え**
   - 3 target 独立取得 → V + DV/LV 競合の 2 seed 固定に変更
   - 閾値段階緩和ループを削除
   - 戻り値を `SeedEntry[]`（可変長）から固定構造に変更

2. **型の更新**
   - `VibrantRole` から競合結果がわかるようにする
   - `AxisInfo.seeds` を固定長（2要素）にするか検討

3. **SVG 表示の更新**
   - 固定 6 seeds のレイアウトに変更
   - 2色目が DV/LV どちらで選ばれたかを表示
