# R4: Neovim Lua ファイル生成・エクスポート

## 概要

R3 が出力した `ContrastResult.adjusted`（`ThemeVariants`）を受け取り、Neovim で `:colorscheme` コマンドで読み込める Lua ファイルを生成・ダウンロードする機能。

### 入出力

| 項目 | 内容 |
|---|---|
| **入力** | `ThemeVariants`（R3 出力 `ContrastResult.adjusted`）＋ テーマメタデータ（名前・キャラクター名） |
| **出力** | Lua ファイル文字列（`string`）→ ブラウザでダウンロード |

### ファイル配置

```
src/features/theme-generator/
  ├── types.ts            # ThemeMeta, ExportFormat 型を追記
  ├── lua-generator.ts    # HighlightMap → Lua 文字列
  └── exporters.ts        # hex / css / json エクスポート + downloadFile
```

### 依存パッケージ

**追加ライブラリなし。** テンプレート文字列（ES6 Template Literals）のみで生成する。

## 型定義

```typescript
// テーマのメタデータ
type ThemeMeta = {
  name: string;           // テーマ名（ファイル名・vim.g.colors_name に使用）
  characterName?: string; // キャラクター名（ファイルヘッダーコメントに使用）
  variant: "dark" | "light";  // 生成するバリアント
};

// エクスポート形式
type ExportFormat = "lua" | "hex" | "css" | "json";
```

## 生成する Lua ファイルの構造

LazyVim ユーザーが `~/.config/nvim/colors/` に置くだけで動作する**単一 `.lua` ファイル**を生成する。

```lua
-- ============================================================
-- oshicolor - {characterName}
-- Generated by https://oshicolor.dev
-- Generated: {date}
-- ============================================================

-- インストール手順:
-- 1. このファイルを ~/.config/nvim/colors/{name}.lua に保存
-- 2. 以下のどちらかの方法で有効化:
--
--    方法 A: init.lua や options.lua に追加
--      vim.cmd.colorscheme("{name}")
--
--    方法 B: LazyVim の lua/plugins/colorscheme.lua に追加
--      return {
--        { "LazyVim/LazyVim", opts = { colorscheme = "{name}" } },
--      }

vim.cmd("highlight clear")
if vim.fn.exists("syntax_on") == 1 then
  vim.cmd("syntax reset")
end

vim.g.colors_name = "{name}"
vim.o.background = "{variant}"

local hi = function(group, opts)
  vim.api.nvim_set_hl(0, group, opts)
end

-- [1] Base colors (palette)
local c = {
  bg      = "{Normal.bg}",
  fg      = "{Normal.fg}",
  comment = "{Comment.fg}",
  string  = "{String.fg}",
  func    = "{Function.fg}",
  keyword = "{Keyword.fg}",
  type    = "{Type.fg}",
  number  = "{Number.fg}",
  special = "{Special.fg}",
  -- UI colors
  cursor_line = "{CursorLine.bg}",
  visual      = "{Visual.bg}",
  pmenu       = "{Pmenu.bg}",
  pmenu_sel   = "{PmenuSel.bg}",
}

-- [2] Editor UI
hi("Normal",        { fg = c.fg, bg = c.bg })
hi("NormalFloat",   { fg = c.fg, bg = c.bg })
hi("CursorLine",    { bg = c.cursor_line })
hi("LineNr",        { fg = c.comment })
hi("CursorLineNr",  { fg = c.fg })
hi("SignColumn",    { bg = c.bg })
hi("VertSplit",     { fg = c.comment })
hi("WinSeparator",  { fg = c.comment })
hi("Visual",        { bg = c.visual })
hi("Search",        { fg = c.bg, bg = c.keyword })
hi("IncSearch",     { fg = c.bg, bg = c.func })
hi("StatusLine",    { fg = c.fg, bg = c.cursor_line })
hi("StatusLineNC",  { fg = c.comment, bg = c.bg })
hi("Pmenu",         { fg = c.fg, bg = c.pmenu })
hi("PmenuSel",      { fg = c.bg, bg = c.pmenu_sel })
hi("PmenuSbar",     { bg = c.pmenu })
hi("PmenuThumb",    { bg = c.comment })

-- [3] Syntax
hi("Comment",   { fg = c.comment, italic = true })
hi("String",    { fg = c.string })
hi("Function",  { fg = c.func })
hi("Keyword",   { fg = c.keyword })
hi("Type",      { fg = c.type })
hi("Number",    { fg = c.number })
hi("Boolean",   { fg = c.number })
hi("Operator",  { fg = c.keyword })
hi("Special",   { fg = c.special })
hi("Constant",  { fg = c.number })
hi("Identifier", { fg = c.fg })
hi("Statement", { fg = c.keyword })
hi("PreProc",   { fg = c.special })
hi("Error",     { fg = c.special, bold = true })

-- [4] Diagnostics
hi("DiagnosticError",            { fg = c.special })
hi("DiagnosticWarn",             { fg = c.func })
hi("DiagnosticInfo",             { fg = c.type })
hi("DiagnosticHint",             { fg = c.comment })
hi("DiagnosticVirtualTextError", { fg = c.special })
hi("DiagnosticVirtualTextWarn",  { fg = c.func })
hi("DiagnosticVirtualTextInfo",  { fg = c.type })
hi("DiagnosticVirtualTextHint",  { fg = c.comment })
hi("DiagnosticUnderlineError",   { undercurl = true, sp = c.special })
hi("DiagnosticUnderlineWarn",    { undercurl = true, sp = c.func })

-- [5] TreeSitter (link to base groups)
-- LazyVim は nvim-treesitter を常に使用するため必須
hi("@variable",           { link = "Identifier" })
hi("@variable.builtin",   { link = "Special" })
hi("@variable.parameter", { link = "Identifier" })
hi("@comment",            { link = "Comment" })
hi("@string",             { link = "String" })
hi("@string.escape",      { link = "Special" })
hi("@function",           { link = "Function" })
hi("@function.builtin",   { link = "Special" })
hi("@function.method",    { link = "Function" })
hi("@function.call",      { link = "Function" })
hi("@keyword",            { link = "Keyword" })
hi("@keyword.function",   { link = "Keyword" })
hi("@keyword.operator",   { link = "Operator" })
hi("@keyword.import",     { link = "PreProc" })
hi("@type",               { link = "Type" })
hi("@type.builtin",       { link = "Type" })
hi("@constant",           { link = "Constant" })
hi("@constant.builtin",   { link = "Special" })
hi("@number",             { link = "Number" })
hi("@boolean",            { link = "Boolean" })
hi("@operator",           { link = "Operator" })
hi("@punctuation.bracket",    { fg = c.fg })
hi("@punctuation.delimiter",  { fg = c.fg })
hi("@property",           { link = "Identifier" })
hi("@module",             { link = "Identifier" })
hi("@tag",                { link = "Keyword" })
hi("@attribute",          { link = "Special" })
```

## ハイライトグループのカテゴリと優先度

| カテゴリ | グループ数 | 優先度 |
|---|---|---|
| エディタ UI | ~15 | MVP 必須 |
| シンタックス（base） | ~10 | MVP 必須 |
| Diagnostics | ~10 | MVP 必須 |
| TreeSitter | ~25（link ベース） | MVP 必須（LazyVim 要件） |
| LSP セマンティックトークン | ~10 | 将来対応 |
| プラグイン専用（Telescope, which-key 等） | ~20 | 将来対応 |

TreeSitter グループは**既存の base グループへの `link`** で定義するため、新しい色を追加する必要はない。フォールバックの仕組みにより `@string.escape` が未定義でも `@string` へ自動フォールバックするが、明示的に定義することで一貫性を保つ。

## 追加エクスポート形式

`exporters.ts` に以下の形式を実装する。すべて `HighlightMap` から生成する。

### Hex リスト（`"hex"`）

```
bg       #0d1117
fg       #e8f4f8
comment  #4a7a8a
string   #39c5bb
...
```

### CSS 変数（`"css"`）

```css
:root {
  --color-bg: #0d1117;
  --color-fg: #e8f4f8;
  --color-comment: #4a7a8a;
  --color-string: #39c5bb;
}
```

### JSON（`"json"`）

```json
{
  "bg": "#0d1117",
  "fg": "#e8f4f8",
  "comment": "#4a7a8a",
  "string": "#39c5bb"
}
```

## ブラウザダウンロード実装

```typescript
// exporters.ts の download ユーティリティ
export const downloadFile = (content: string, filename: string, mimeType: string): void => {
  const blob = new Blob([content], { type: mimeType });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = filename;
  a.click();
  URL.revokeObjectURL(url);
};

// Lua ダウンロード
downloadFile(luaContent, `${meta.name}.lua`, "text/plain");
```

## 検証方法

| 検証項目 | 手順 |
|---|---|
| **構文確認** | 生成された Lua ファイルを `luac -p` でパース（構文エラーがないか） |
| **実機確認** | `~/.config/nvim/colors/oshicolor.lua` に置いて LazyVim 上で `:colorscheme oshicolor` が通ることを確認 |
| **TreeSitter 確認** | `:TSInstall` 後に TypeScript ファイルを開き、`@function`, `@keyword`, `@string` が正しく色付けされることを確認 |
| **Diagnostic 確認** | LSP が有効な状態でエラー・警告が正しく色付けされることを確認 |
