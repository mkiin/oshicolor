# R1/V1 色抽出プラン

## 初期方針: 貪欲法 + OKLab 知覚距離

### Context

`requipments-phase1.md` の R1「画像アップロード・色抽出」は「k-means または中央値カット法」と記述されていたが、参考実装（anicolors）の解析とライブラリ調査により、より具体的かつ改善された実装方針が確立できた。

### 参考実装との差分（改善点）

#### 懸念点 1: RGB ユークリッド距離は知覚的に不正確

参考実装の貪欲法は `sqrt((r1-r2)² + (g1-g2)² + (b1-b2)²)` で距離を計算している。
RGB 空間は人間の知覚と一致しないため、アニメ画像では「RGB 上は離れているが視覚的には似て見える」色が選ばれる場合がある。

**改善**: OKLab 色空間での知覚的距離（`culori` の `differenceCiede2000` 等）を使用する。

#### 懸念点 2: ライブラリの老朽化

| ライブラリ             | 状況                | 対応                              |
| ---------------------- | ------------------- | --------------------------------- |
| `nearest-color` v0.4.4 | 7年前に開発停止     | `culori` + 独自実装に置き換え     |
| `quantize` v1.0.2      | 開発停止の可能性    | MVP は不使用（貪欲法で代替）      |
| `react-color` v2.19.3  | 5年前に開発停止     | `react-colorful` に置き換え（R5） |
| `color` v5.0.0         | 継続中              | `culori` に統合できるため不要     |

#### 懸念点 3: `culori` 1 本で R1・R3 を横断できる

- R1: RGB → HEX 変換、OKLab 距離計算
- R3: OKLch の L 値（明度）操作、コントラスト比計算

`color` ライブラリを `culori` に統一することで依存関係を削減できる。

#### 懸念点 4: 座標系が 384 固定値でハードコード

参考実装は正規化座標を `0〜384` スケールで管理している。oshicolor では **0〜1 正規化** に変更。

### アルゴリズム

1. Canvas に画像を原寸描画し `getImageData()` で全ピクセル取得
2. 8px 間隔でサンプリング、透明（alpha < 128）・極端な明度（30未満/225超）を除外
3. RGB 各チャンネルを 24 単位で量子化してグルーピング、出現回数 ≥ 10 の上位 20 色を候補に
4. 貪欲法で多様性最大化：既選択色との **OKLab 知覚距離** が最大の色を順次選択
5. 各色に `culori` で HEX 変換し、`color-name-list` + 独自 nearest 検索で色名付与

### 依存パッケージ

| パッケージ      | バージョン | 用途                                          |
| --------------- | ---------- | --------------------------------------------- |
| `culori`        | ^4.0.2     | OKLab/OKLch 色操作・距離計算（R1 + R3 共通）  |
| `color-name-list` | ^14.x   | 色名データベース                              |

---

## 方針変更: extract-colors へ移行

### 経緯

貪欲法実装から `extract-colors` ライブラリベースの実装に移行した。
一時期 OKLab / Cinematic の2実装比較構成を取っていたが、差異がほぼなくなったため1本化。

### 廃止したライブラリ

| ライブラリ      | 経緯                                                                                                                       |
| --------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `quantize`      | 中央値カット法（MMCQ）実装として採用。`var` スコープバグ・PQueue 不安定ソート・8色超で要求数−1しか返らないバグが判明し廃止 |
| `nearest-color` | Cinematic 実装の色名解決に採用。OKLab 実装に統一したため不要になり廃止                                                     |

### アルゴリズムの変更

| 項目           | 貪欲法                                       | extract-colors                                      |
| -------------- | -------------------------------------------- | --------------------------------------------------- |
| 色抽出         | 独自実装（量子化 → 候補絞り込み → 貪欲選択） | `extract-colors` ライブラリ                         |
| サンプリング   | SAMPLE_STEP=4px 固定ループ                   | `pixels: 40_000`（ライブラリが reducer を自動計算） |
| クラスタリング | RGB 量子化（24 単位丸め）+ グループ集計      | RGB 距離マージ → HSL 距離マージの2段階              |
| 色選択         | 貪欲法（OKLab 距離で多様性最大化）           | `area` 降順スライス                                 |
| 並び順         | 最頻出色 → OKLab 距離最遠順                  | 面積（area）降順                                    |
| 色名解決       | OKLab 距離（culori）                         | OKLab 距離（culori）← 変更なし                      |

### 既知の制約

- **色数を直接指定できない**: `extract-colors` は `distance` 等のパラメータで間接的に色数を制御する設計。12 色を下回る場合がある
- **`x` / `y` 座標が常に 0**: ライブラリが位置情報を返さないため、位置ベースの判断ができない
