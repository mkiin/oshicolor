# oshicolor

## 何をするアプリか

キャラクターイラストや任意の画像から、Neovim 用カラースキーム（Lua 形式）を生成する Web アプリケーション。色抽出からハイライトグループへの割り当て、コントラスト自動調整までを一貫して行い、ダーク/ライト両バリアントとブラウザプレビューを提供する。「パレット抽出 → 手書き Lua」の従来 2 段階ワークフローを 1 ステップに圧縮することが核となる価値。

## 誰のためか

推しキャラクターのビジュアルから開発環境のカラースキームを作りたい Neovim ユーザー。dotfiles・ricing 文化に親しむ層を主対象とする。「推し色でエディタを染める」という Emotional Job と「最小手順で実用テーマを得る」という Functional Job を同時に満たす。Doki Theme（プリセット中心・Neovim 未対応）や AniColors（エディタ出力なし）といった近接プロダクトはあるが、画像入力から Neovim テーマ出力までを繋ぐソリューションは現状存在しない。

## スコープ

MVP は Web アプリのみ・Neovim（Lua）のみを対象とする。コントラスト保証は WCAG を参考にした自動調整、ハイライト割り当ては OKLch ベースのルールで実装する。画像処理はクライアントサイドで完結させ、ユーザー画像はサーバーに送信しない。VSCode・ターミナル向けエクスポート、ネイティブアプリ、コミュニティギャラリー、複数画像対応は MVP 後の検討事項。
