## @swc-jotai/react-refresh

このプラグインは Jotai アトムに対する React Refresh のサポートを追加します。React Refresh を使用して開発する際に状態がリセットされないことを保証します。

### 使用方法

次のコマンドでインストールします：

```sh
npm install --save-dev @swc-jotai/react-refresh
```

プラグインを `.swcrc` に追加できます：

```json
{
  "jsc": {
    "experimental": {
      "plugins": [["@swc-jotai/react-refresh", {}]]
    }
  }
}
```

Next.js の [実験的な SWC プラグイン機能](https://nextjs.org/docs/advanced-features/compiler#swc-plugins-experimental) でプラグインを使用できます。

```js
module.exports = {
  experimental: {
    swcPlugins: [['@swc-jotai/react-refresh', {}]],
  },
}
```

## @swc-jotai/debug-label

Jotai はキー（Recoil のような）ではなくオブジェクト参照に基づいています。そのため、アトムの識別子がありません。アトムを識別するために、React devtools で確認できる `debugLabel` をアトムに追加することが可能です。

しかし、すべてのアトムに `debugLabel` を追加するのはすぐに煩雑になります。

この `SWC` プラグインは識別子に基づいて各アトムに `debugLabel` を自動的に追加します。

プラグインは以下のコードを変換します：

```js
export const countAtom = atom(0)
```

以下のように変換されます：

```js
export const countAtom = atom(0)
countAtom.debugLabel = 'countAtom'
```

デフォルトエクスポートもファイル名に基づいて処理されます：

```js
// countAtom.ts
export default atom(0)
```

以下のように変換されます：

```js
// countAtom.ts
const countAtom = atom(0)
countAtom.debugLabel = 'countAtom'
export default countAtom
```

### 使用方法

次のコマンドでインストールします：

```sh
npm install --save-dev @swc-jotai/debug-label
```

プラグインを `.swcrc` に追加できます：

```json
{
  "jsc": {
    "experimental": {
      "plugins": [["@swc-jotai/debug-label", {}]]
    }
  }
}
```

あるいは Next.js の [実験的な SWC プラグイン機能](https://nextjs.org/docs/advanced-features/compiler#swc-plugins-experimental) でプラグインを使用できます。

```js
module.exports = {
  experimental: {
    swcPlugins: [['@swc-jotai/debug-label', {}]],
  },
}
```

以下に例があります。

### カスタムアトム名

カスタムアトムに対してプラグインを有効にできます。以下のようにプラグインに指定します：

```js
module.exports = {
  experimental: {
    swcPlugins: [
      ['@swc-jotai/debug-label', { atomNames: ['customAtom'] }],
      ['@swc-jotai/react-refresh', { atomNames: ['customAtom'] }],
    ],
  },
}
```
