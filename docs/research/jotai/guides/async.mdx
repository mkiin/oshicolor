非同期アトムを使用すると、アトムから直接管理しながら、信じられないほど簡単に実際のデータにアクセスできます。

大きく2つのカテゴリに分けられます：

- 非同期 read アトム：値を取得しようとした瞬間に非同期リクエストが開始されます。「スマートゲッター」として考えることができます。
- 非同期 write アトム：特定のタイミングで非同期リクエストが開始されます。「アクション」として考えることができます。

## 非同期 read アトム

アトムの `read` 関数は Promise を返すことができます。

```js
const countAtom = atom(1)
const asyncAtom = atom(async (get) => get(countAtom) * 2)
```

Jotai は本質的に `Suspense` を活用して非同期フローを処理します。

```jsx
const ComponentUsingAsyncAtoms = () => {
  const [num] = useAtom(asyncAtom)
  // ここで `num` は asyncAtom が Promise を返すにもかかわらず、常に `number` です
}
const App = () => {
  return (
    <Suspense fallback={/* サスペンド中に表示するもの */}>
      <ComponentUsingAsyncAtoms />
    </Suspense>
  )
}
```

あるいは、アトムを [`loadable` API](../utilities/async.mdx) でラップすることで、Jotai が行う固有のサスペンドを避けることができます。

別のアトムが非同期アトムを使用する場合、Promise を返します。そのため、そのアトムも非同期にする必要があります。

```js
const anotherAtom = atom(async (get) => (await get(asyncAtom)) / 2)
```

これは write 関数を持つアトムにも適用されます。

```js
const asyncAtom = atom(async (get) => ...)
const writeAtom = atom(null, async (get, set, payload) => {
  await get(asyncAtom)
  // ...
})
```

## 非同期 write アトム

非同期 write アトムは別の種類の非同期アトムです。アトムの `write` 関数が Promise を返す場合です。

```js
const countAtom = atom(1)
const asyncIncrementAtom = atom(null, async (get, set) => {
  // 何かを await する
  set(countAtom, get(countAtom) + 1)
})

const Component = () => {
  const [, increment] = useAtom(asyncIncrementAtom)
  const handleClick = () => {
    increment()
  }
  // ...
}
```

## 時々非同期

Jotai で実現できる興味深いパターンは、必要なときにサスペンドをトリガーするために非同期から同期に切り替えることです。

```js
const request = async () => fetch('https://...').then((res) => res.json())
const baseAtom = atom(0)
const Component = () => {
  const [value, setValue] = useAtom(baseAtom)
  const handleClick = () => {
    setValue(request()) // リクエストが解決するまでサスペンドします
  }
  // ...
}
```

### TypeScript での使用

TypeScript では `atom(0)` は `PrimitiveAtom<number>` として推論されます。`Promise<number>` を値として受け入れることができないため、上記のコードは型チェックを通過しません。これに対応するには、アトムを明示的に型付けして `Promise<number>` を許容値として追加する必要があります。

```ts
const baseAtom = atom<number | Promise<number>>(0) // 同期・非同期の両方の値を受け入れます
```

## 永遠に非同期

不確定な時点（または永遠に）までサスペンドしたい場合があります。

```js
const baseAtom = atom(new Promise(() => {})) // 別途セットされるまでサスペンドします
```

## Suspense

非同期サポートは Jotai のファーストクラス機能です。コアで React Suspense を完全に活用します。

> 技術的には、React.lazy 以外での Suspense の使用は React 17 では未サポート・未ドキュメントです。これが問題になる場合でも、サスペンドを避けるために [`loadable` API](../utilities/async.mdx) を使用できます。

非同期アトムを使用するには、コンポーネントツリーを `<Suspense>` でラップする必要があります。

> `<Provider>` がある場合は、その `<Provider>` 内に**少なくとも1つ**の `<Suspense>` を配置してください。そうしないと、コンポーネントレンダリング中に無限ループが発生する可能性があります。

```jsx
const App = () => (
  <Provider>
    <Suspense fallback="Loading...">
      <Layout />
    </Suspense>
  </Provider>
)
```

コンポーネントツリーに複数の `<Suspense>` を持つことも可能であり、Jotai の組み込み処理を最大限に活用するために検討すべきです。
