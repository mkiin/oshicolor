Jotai は React 開発の基本に立ち返り、すべてをシンプルに保つためのライブラリです。

### ゼロから

Jotai を以前知っていたものと比較する前に、非常にシンプルなことに直接飛び込んでみましょう。

React の世界は私たちの世界に非常に似ています。コンポーネントと呼ばれる小さなエンティティの大きな集合であり、それぞれが独自の状態を持っています。コンポーネントを構造化して互いに連携させることでアプリが作られます。

Jotai の世界にも小さなエンティティ、アトムがあり、それらも独自の状態を持っています。アトムを組み合わせることでアプリの状態が作られます！

Jotai はあらゆるものをアトムと見なします。「オブジェクトや配列が必要で、それらをフィルタリングしてソートしたい」と思うかもしれません。
そしてここに Jotai の美しさがあります。Jotai はより単純なアトムから派生したシンプルなアトムを優雅に作成できます。

例えば、オンラインの友達とオフラインの友達という2つのタブを持つページがある場合、共通のよりシンプルなソースから単純に派生した2つのアトムを持つことになります。

```js
const dumbAtom = atom([{ name: 'Friend 1', online: false }])
const onlineAtom = atom((get) => get(dumbAtom).filter((item) => item.online))
const offlineAtom = atom((get) => get(dumbAtom).filter((item) => !item.online))
```

複雑さはいくらでも増やすことができます。

Jotai のもう一つの素晴らしい機能は、非同期アトムを使用するときに組み込みで Suspend できる能力です。これは比較的新しい機能でさらなる実験が必要ですが、確実に React アプリを構築する方法の未来です。詳細は[ドキュメント](https://react.dev/blog/2022/03/29/react-v18#new-suspense-features)を確認してください。
