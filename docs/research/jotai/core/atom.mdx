## atom

`atom` 関数はアトムコンフィグを作成するためのものです。
値をまだ保持していない定義であるため「アトムコンフィグ」と呼びます。
文脈が明確な場合は単に「アトム」とも呼びます。

アトムコンフィグはイミュータブルなオブジェクトです。アトムコンフィグオブジェクトは値を保持しません。アトムの値はストアに存在します。

プリミティブアトム（コンフィグ）を作成するには、初期値を提供するだけです。

```js
import { atom } from 'jotai'

const priceAtom = atom(10)
const messageAtom = atom('hello')
const productAtom = atom({ id: 12, name: 'good stuff' })
```

派生アトムも作成できます。3つのパターンがあります：

- 読み取り専用アトム
- 書き込み専用アトム
- 読み書き可能アトム

派生アトムを作成するには、read 関数とオプションの write 関数を渡します。

```js
const readOnlyAtom = atom((get) => get(priceAtom) * 2)
const writeOnlyAtom = atom(
  null, // 第1引数に `null` を渡すのが慣例です
  (get, set, update) => {
    // `update` はこのアトムを更新するために受け取る任意の単一値です
    set(priceAtom, get(priceAtom) - update.discount)
    // または第2引数に関数を渡すこともできます
    // その関数が呼び出され、
    //  アトムの現在値が第1引数として渡されます
    set(priceAtom, (price) => price - update.discount)
  },
)
const readWriteAtom = atom(
  (get) => get(priceAtom) * 2,
  (get, set, newPrice) => {
    set(priceAtom, newPrice / 2)
    // 同時に好きなだけアトムをセットできます
  },
)
```

read 関数の `get` はアトムの値を読み取るためのものです。
リアクティブであり、読み取りの依存関係が追跡されます。

write 関数の `get` もアトムの値を読み取りますが、追跡されません。
さらに、Jotai v1 API では未解決の非同期値を読み取ることができません。

write 関数の `set` はアトムの値を書き込むためのものです。
対象アトムの write 関数を呼び出します。

### render 関数内でのアトム作成に関する注意

アトムコンフィグはどこでも作成できますが、参照等価性が重要です。
動的に作成することもできます。
render 関数内でアトムを作成するには、安定した参照を得るために `useMemo` または `useRef` が必要です。`useMemo` と `useRef` のどちらを使うか迷った場合は `useMemo` を使用してください。
そうしないと、`useAtom` で無限ループが発生する可能性があります。

```js
const Component = ({ value }) => {
  const valueAtom = useMemo(() => atom({ value }), [value])
  // ...
}
```

### シグネチャ

```ts
// プリミティブアトム
function atom<Value>(initialValue: Value): PrimitiveAtom<Value>

// 読み取り専用アトム
function atom<Value>(read: (get: Getter) => Value): Atom<Value>

// 書き込み可能な派生アトム
function atom<Value, Args extends unknown[], Result>(
  read: (get: Getter) => Value,
  write: (get: Getter, set: Setter, ...args: Args) => Result,
): WritableAtom<Value, Args, Result>

// 書き込み専用の派生アトム
function atom<Value, Args extends unknown[], Result>(
  read: Value,
  write: (get: Getter, set: Setter, ...args: Args) => Result,
): WritableAtom<Value, Args, Result>
```

- `initialValue`: 値が変更されるまでアトムが返す初期値。
- `read`: アトムが読み取られるたびに評価される関数。`read` のシグネチャは `(get) => Value` で、`get` はアトムコンフィグを受け取り、以下に説明するように Provider に格納されたその値を返す関数です。依存関係が追跡されるため、`get` が少なくとも1回アトムに使用されると、アトムの値が変更されるたびに `read` が再評価されます。
- `write`: 主にアトムの値を変更するために使用される関数。`useAtom` が返すペアの第2値 `useAtom()[1]` を呼び出すたびに呼び出されます。プリミティブアトムにおけるこの関数のデフォルト値はそのアトムの値を変更します。`write` のシグネチャは `(get, set, ...args) => Result` です。`get` は上述のものと同様ですが、依存関係を追跡しません。`set` はアトムコンフィグと新しい値を受け取り、Provider のアトム値を更新する関数です。`...args` は `useAtom()[1]` を呼び出す際に受け取る引数です。`Result` は `write` 関数の戻り値です。

```js
const primitiveAtom = atom(initialValue)
const derivedAtomWithRead = atom(read)
const derivedAtomWithReadWrite = atom(read, write)
const derivedAtomWithWriteOnly = atom(null, write)
```

アトムには書き込み可能なアトムと読み取り専用アトムの2種類があります。プリミティブアトムは常に書き込み可能です。派生アトムは `write` が指定されていれば書き込み可能です。プリミティブアトムの `write` は `React.useState` の `setState` と同等です。

### `debugLabel` プロパティ

作成されたアトムコンフィグはオプションのプロパティ `debugLabel` を持てます。デバッグラベルはデバッグ時にアトムを表示するために使用されます。詳細は[デバッグガイド](../guides/debugging.mdx)を参照してください。

注意：デバッグラベルは一意である必要はありませんが、識別可能にすることを一般的に推奨します。

### `onMount` プロパティ

作成されたアトムコンフィグはオプションのプロパティ `onMount` を持てます。`onMount` は `setAtom` 関数を受け取り、オプションで `onUnmount` 関数を返す関数です。

`onMount` 関数はアトムが初めて Provider でサブスクライブされたときに呼び出され、
`onUnmount` はサブスクライブされなくなったときに呼び出されます。
一部のケース（[React strict mode](https://react.dev/reference/react/StrictMode) など）では、
アトムがアンマウントされた直後に再マウントされることがあります。

```js
const anAtom = atom(1)
anAtom.onMount = (setAtom) => {
  console.log('atom is mounted in provider')
  setAtom(c => c + 1) // マウント時にカウントをインクリメント
  return () => { ... } // オプションの onUnmount 関数を返す
}

const Component = () => {
  // 以下のケースでは、コンポーネントのマウント時に `onMount` が呼び出されます：
  useAtom(anAtom)
  useAtomValue(anAtom)

  // しかし、以下のケースではアトムがサブスクライブされないため、
  //  `onMount` は呼び出されません：
  useSetAtom(anAtom)
  useAtomCallback(
    useCallback((get) => get(anAtom), []),
  )
  // ...
}
```

`setAtom` 関数を呼び出すとアトムの `write` が呼び出されます。`write` をカスタマイズすることで動作を変更できます。

```js
const countAtom = atom(1)
const derivedAtom = atom(
  (get) => get(countAtom),
  (get, set, action) => {
    if (action.type === 'init') {
      set(countAtom, 10)
    } else if (action.type === 'inc') {
      set(countAtom, (c) => c + 1)
    }
  },
)
derivedAtom.onMount = (setAtom) => {
  setAtom({ type: 'init' })
}
```

### 高度な API

Jotai v2 以降、`read` 関数は第2引数 `options` を持ちます。

#### `options.signal`

[AbortController](https://developer.mozilla.org/en-US/docs/Web/API/AbortController) を使用して、非同期関数を中止できます。
中止は新しい計算（`read` 関数の呼び出し）が開始される前にトリガーされます。

使用方法：

```ts
const readOnlyDerivedAtom = atom(async (get, { signal }) => {
  // signal を使用して関数を中止する
})

const writableDerivedAtom = atom(
  async (get, { signal }) => {
    // signal を使用して関数を中止する
  },
  (get, set, arg) => {
    // ...
  },
)
```

`signal` の値は [AbortSignal](https://developer.mozilla.org/en-US/docs/Web/API/AbortSignal) です。
`signal.aborted` のboolean値を確認するか、`addEventListener` で `abort` イベントを使用できます。

`fetch` のユースケースでは、単純に `signal` を渡せます。

`fetch` の使用例については以下のサンプルを参照してください。

#### `options.setSelf`

自身のアトムの write 関数を呼び出す特別な関数です。

⚠️ 主に内部使用およびサードパーティライブラリ作者向けに提供されています。動作を理解するためにソースコードを注意深く読んでください。破壊的・非破壊的変更についてはリリースノートを確認してください。

```tsx
import { Suspense } from 'react'
import { atom, useAtom } from 'jotai'

const userIdAtom = atom(1)
const userAtom = atom(async (get, { signal }) => {
  const userId = get(userIdAtom)
  const response = await fetch(
    `https://jsonplaceholder.typicode.com/users/${userId}?_delay=2000`,
    { signal },
  )
  return response.json()
})

const Controls = () => {
  const [userId, setUserId] = useAtom(userIdAtom)
  return (
    <div>
      User Id: {userId}
      <button onClick={() => setUserId((c) => c - 1)}>Prev</button>
      <button onClick={() => setUserId((c) => c + 1)}>Next</button>
    </div>
  )
}

const UserName = () => {
  const [user] = useAtom(userAtom)
  return <div>User name: {user.name}</div>
}

const App = () => (
  <>
    <Controls />
    <Suspense fallback="Loading...">
      <UserName />
    </Suspense>
  </>
)

export default App
```
