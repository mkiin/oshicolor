React での Jotai の使用を拡張するライブラリがいくつかあります。

## `jotai-scope`

Jotai の Provider はサブツリー下で Jotai のストアをスコープできますが、
サブツリー内でそのツリーより上のストアは使用できません。

回避策として、useAtom や他のフックで `store` オプションを使用する方法があります。

`store` オプションを指定する代わりに、
`<ScopeProvider>` は React ツリーの異なる部分で**状態を共有せずに**同じアトムを再利用しつつ、
親ストアから他のアトムを読み取ることができます。

### インストール

```
npm install jotai-scope
```

### カウンターの例

```tsx
import { atom, useAtom } from "jotai";
import { ScopeProvider } from "jotai-scope";

const countAtom = atom(0);

function Counter() {
  const [count, setCount] = useAtom(countAtom);
  const [anotherCount, setAnotherCount] = useAtom(anotherCountAtom);
  return (
    <>
      <div>
        <span>count: {count}</span>
        <button type="button" onClick={() => setCount((v) => v + 1)}>
          increment
        </button>
      </div>
      <div>
        <span>another count: {anotherCount}</span>
        <button type="button" onClick={() => setAnotherCount((v) => v + 1)}>
          increment
        </button>
      </div>
    </>
  );
}

function App() {
  return (
    <div>
      <Counter />
      <ScopeProvider atoms={[countAtom]}>
        <Counter />
      </ScopeProvider>
    </div>
  );
}
```

## `createIsolation`

Jotai の Provider と `jotai-scope` のスコープされた Provider の両方は、
まだグローバルコンテキストを使用しています。

Jotai に依存するライブラリを開発していて、
ライブラリのユーザーがアプリで Jotai を別途使用する可能性がある場合、
同じコンテキストを共有する可能性があります。これにより、予期しない Jotai ストアを指す可能性があり、問題が発生することがあります。

コンテキストの競合を避けるために、
`jotai-scope` から `createIsolation` というユーティリティ関数がエクスポートされています。

```tsx
import { createIsolation } from "jotai-scope";

const { Provider, useStore, useAtom, useAtomValue, useSetAtom } = createIsolation();

function Library() {
  return (
    <Provider>
      <LibraryComponent />
    </Provider>
  );
}
```

## `bunshi`（旧 `jotai-molecules`）

Jotai アトムは再レンダリングを最適化するための基本的なソリューションを提供します。
グローバルに定義されたアトムは他のアトムに依存できますが、
コンポーネントツリー内の props や状態には依存できません。
コンポーネントツリー内でアトムを定義することは可能ですが、
それらのアトムを何らかの方法で渡す必要があります
（例：[atoms-in-atom](../guides/atoms-in-atom.mdx)）。

[bunshi](https://github.com/saasquatch/bunshi) はそのようなユースケースを支援するサードパーティライブラリです。

詳細については [モチベーション](https://github.com/saasquatch/bunshi/tree/v1.1.1#motivation) を参照してください。

### インストール

```
npm install bunshi
```

### カウンターの例

```jsx
import { atom, useAtom } from "jotai";
import { molecule, useMolecule, createScope, ScopeProvider } from "bunshi/react";

const InitialCountScope = createScope({ initialCount: 0 });
const countMolecule = molecule((getMol, getScope) => {
  const { initialCount } = getScope(InitialCountScope);
  return atom(initialCount);
});

function Counter() {
  const countAtom = useMolecule(countMolecule);
  const [count, setCount] = useAtom(countAtom);
  return (
    <div>
      {count} <button onClick={() => setCount((v) => v + 1)}>+1</button>
    </div>
  );
}

function App() {
  return (
    <div>
      <h1>With initial value 1</h1>
      <ScopeProvider scope={InitialCountScope} value={{ initialCount: 1 }}>
        <Counter />
        <Counter />
      </ScopeProvider>
      <h1>With initial value 2</h1>
      <ScopeProvider scope={InitialCountScope} value={{ initialCount: 2 }}>
        <Counter />
        <Counter />
      </ScopeProvider>
      <h1>Default</h1>
      <Counter />
      <Counter />
    </div>
  );
}
```
