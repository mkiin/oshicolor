[urql](https://formidable.com/open-source/urql/) は GraphQL クエリ、キャッシュ、状態管理のためのツールキットを提供します。

[概要ドキュメント](https://formidable.com/open-source/urql/docs/) より：

> urql は高度にカスタマイズ可能で汎用的な GraphQL クライアントで、成長するにつれて正規化キャッシュなどの機能を追加できます。GraphQL 初心者にとっても使いやすく、動的なシングルページアプリケーションや高度にカスタマイズされた GraphQL インフラストラクチャをサポートするために拡張可能です。要するに、urql は使いやすさと適応性を優先しています。

[jotai-urql](https://github.com/jotaijs/jotai-urql) は URQL のための Jotai 拡張ライブラリです。URQL の GraphQL 機能をすべて取り込んだ統一されたインターフェースを提供し、既存の Jotai 状態と組み合わせてこれらの機能を活用できます。

### インストール

この拡張を使用するには `jotai-urql`、`@urql/core`、`wonka` をインストールする必要があります。

```
npm install jotai-urql @urql/core wonka
```

### エクスポートされる関数

- `atomWithQuery` は [client.query](https://formidable.com/open-source/urql/docs/api/core/#clientquery) 用
- `atomWithMutation` は [client.mutation](https://formidable.com/open-source/urql/docs/api/core/#clientmutation) 用
- `atomWithSubscription` は [client.subscription](https://formidable.com/open-source/urql/docs/api/core/#clientsubscription) 用

### 基本的な使用方法

#### クエリ：

```tsx
import { useAtom } from "jotai";

const countQueryAtom = atomWithQuery<{ count: number }>({
  query: "query Count { count }",
  getClient: () => client, // このオプションは `useRehydrateAtom([[clientAtom, client]])` がグローバルに使用されている場合はオプションです
});

const Counter = () => {
  // 最初のオペレーション結果が解決されるまでサスペンドします。エラー、部分データ、データのいずれかで解決されます
  const [operationResult, reexecute] = useAtom(countQueryAtom);

  if (operationResult.error) {
    // これは上位の ErrorBoundary で処理する必要があります
    throw operationResult.error;
  }

  // データはこの時点で undefined の可能性があるため（エラーの場合のみ）、オプショナルチェーンを使用する必要があります
  return <>{operationResult.data?.count}</>;
};
```

#### ミューテーション：

```tsx
import { useAtom } from "jotai";

const incrementMutationAtom = atomWithMutation<{ increment: number }>({
  query: "mutation Increment { increment }",
});

const Counter = () => {
  const [operationResult, executeMutation] = useAtom(incrementMutationAtom);
  return (
    <div>
      <button onClick={() => executeMutation().then((it) => console.log(it.data?.increment))}>
        Increment
      </button>
      <div>{operationResult.data?.increment}</div>
    </div>
  );
};
```

### 関数に渡されるオプションの簡略化された型

```tsx
type AtomWithQueryOptions<Data = unknown, Variables extends AnyVariables = AnyVariables> = {
  // 文字列クエリ、typed-document-node、document node などをサポート
  query: DocumentInput<Data, Variables>;
  // ジェネリック/typed-document-node の型に基づいて動的に強制されます
  getVariables?: (get: Getter) => Variables;
  getContext?: (get: Getter) => Partial<OperationContext>;
  getPause?: (get: Getter) => boolean;
  getClient?: (get: Getter) => Client;
};

type AtomWithMutationOptions<Data = unknown, Variables extends AnyVariables = AnyVariables> = {
  query: DocumentInput<Data, Variables>;
  getClient?: (get: Getter) => Client;
};

// サブスクリプションの型は AtomWithQueryOptions と同じです
```

### サスペンスを無効にする

より安定していることが証明されているため、代わりに `import { loadable } from "jotai/utils"` の使用が推奨されます。ただし、それでも無効にしたい場合は以下のように行います：

```tsx
import { suspenseAtom } from "jotai-urql";

export const App = () => {
  // アプリ全体のサスペンスを無効にします
  useHydrateAtoms([[suspenseAtom, false]]);
  return <Counter />;
};
```

### 便利なヘルパーフック

以下はヘルパーフックです。まれなコーナーケースをカバーし、エラーが（Promise の reject の場合に）エラーとして扱われ、主に近くの ErrorBoundaries で処理される `@tanstack/react-query` のデフォルト動作に似た使用方法にします。サスペンドバージョンのみで有効です。

#### useQueryAtomData

解決後に `data` をきれいに返し、すべてのエラースロー/再実行ケース/コーナーケースを処理します。
型がオーバーライドされているため、`data` はクエリ自体の期待する戻り型でない限り、`undefined` にも `null` にもなりません。

```tsx
import type { AnyVariables, OperationResult } from "@urql/core";
import { useAtom } from "jotai";
import type { AtomWithQuery } from "jotai-urql";

export const useQueryAtomData = <Data = unknown, Variables extends AnyVariables = AnyVariables>(
  queryAtom: AtomWithQuery<Data, Variables>,
) => {
  const [opResult, dispatch] = useAtom(queryAtom);

  if (opResult.error && opResult.stale) {
    use(
      // ここでツリーをサスペンドします。これは ErrorBoundary リトライロジックで
      // `network-only` を使用する場合のみトリガーされます。その場合ツリーがサスペンドせず、
      // "スロー - バウンダリでリトライ - スロー - バウンダリでリトライ" のサイクルが発生する可能性があります
      // (Jotai URQL バインディングのみ)。
      // eslint-disable-next-line promise/avoid-new
      new Promise((resolve) => {
        setTimeout(resolve, 10000); // このタイムアウト時間は新しいオペレーション結果が来るまで
        // このコンポーネントをサスペンドし続けます。10秒後には単純にコンポーネント自体のレンダリングを試み、
        // 無限ループで再びサスペンドします
      }),
    );
  }

  if (opResult.error) {
    throw opResult.error;
  }

  if (!opResult.data) {
    throw Error(
      "Query data is undefined. Probably you paused the query? In that case use `useQueryAtom` instead.",
    );
  }
  return [opResult.data, dispatch, opResult] as [
    Exclude<typeof opResult.data, undefined>,
    typeof dispatch,
    typeof opResult,
  ];
};

// Promise が解決中にツリーをサスペンド（React の次のバージョンでは不要になります）
function use(promise: Promise<any> | any) {
  if (promise.status === "fulfilled") {
    return promise.value;
  }
  if (promise.status === "rejected") {
    throw promise.reason;
  } else if (promise.status === "pending") {
    throw promise;
  } else {
    promise.status = "pending";
    // eslint-disable-next-line promise/catch-or-return
    (promise as Promise<any>).then(
      (result: any) => {
        promise.status = "fulfilled";
        promise.value = result;
      },
      (reason: any) => {
        promise.status = "rejected";
        promise.reason = reason;
      },
    );
    throw promise;
  }
}
```

### アトムと urql プロバイダーで同じクライアントインスタンスを参照する

同じ urqlClient オブジェクトを参照するために、プロジェクトのルートを `<Provider>` でラップし、UrqlProvider に渡したのと同じ urqlClient 値で clientAtom を初期化してください。

この手順なしでは、`atomWithQuery` を使用するたびにクライアントを指定しなければならない場合があります。これにより、オプションの `getClient` パラメータを無視して、コンテキストからクライアントを使用できるようになります。

```jsx
import { Suspense } from "react";
import { Provider } from "jotai/react";
import { useHydrateAtoms } from "jotai/react/utils";
import { clientAtom } from "jotai-urql";

import { createClient, cacheExchange, fetchExchange, Provider as UrqlProvider } from "urql";

const urqlClient = createClient({
  url: "https://countries.trevorblades.com/",
  exchanges: [cacheExchange, fetchExchange],
  fetchOptions: () => {
    return { headers: {} };
  },
});

const HydrateAtoms = ({ children }) => {
  useHydrateAtoms([[clientAtom, urqlClient]]);
  return children;
};

export default function MyApp({ Component, pageProps }) {
  return (
    <UrqlProvider value={urqlClient}>
      <Provider>
        <HydrateAtoms>
          <Suspense fallback="Loading...">
            <Component {...pageProps} />
          </Suspense>
        </HydrateAtoms>
      </Provider>
    </UrqlProvider>
  );
}
```
