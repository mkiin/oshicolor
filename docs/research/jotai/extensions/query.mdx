[TanStack Query](https://tanstack.com/query/) は非同期状態（通常は外部データ）を管理するための関数セットを提供します。

[概要ドキュメント](https://tanstack.com/query/v5/docs/framework/react/overview) より：

> React Query は React の欠けているデータフェッチライブラリとよく言われますが、より技術的な表現では、React アプリケーションでの**サーバー状態のフェッチ、キャッシュ、同期、更新**を簡単にします。

[jotai-tanstack-query](https://github.com/jotai-labs/jotai-tanstack-query) は TanStack Query のための Jotai 拡張ライブラリです。TanStack Query の全機能を備えた素晴らしいインターフェースを提供し、既存の Jotai 状態と組み合わせてこれらの機能を使用できます。

### サポート

jotai-tanstack-query は現在 TanStack Query v5 をサポートしています。

### インストール

`jotai` に加えて、この拡張を使用するには `jotai-tanstack-query` と `@tanstack/query-core` をインストールする必要があります。

```bash
npm install jotai-tanstack-query @tanstack/query-core
```

### 段階的な導入

`jotai-tanstack-query` をアプリに段階的に導入できます。すべてまたは何もしないという解決策ではありません。同じ QueryClient インスタンスを使用することを確認するだけです。[QueryClient のセットアップ](#referencing-the-same-instance-of-query-client-in-your-project)。

```jsx
// 既存の useQueryHook
const { data, isPending, isError } = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
})

// jotai-tanstack-query
const todosAtom = atomWithQuery(() => ({
  queryKey: ['todos'],
}))

const [{ data, isPending, isError }] = useAtom(todosAtom)
```

### エクスポートされる関数

- `atomWithQuery` は [useQuery](https://tanstack.com/query/v5/docs/react/reference/useQuery) 用
- `atomWithInfiniteQuery` は [useInfiniteQuery](https://tanstack.com/query/v5/docs/react/reference/useInfiniteQuery) 用
- `atomWithMutation` は [useMutation](https://tanstack.com/query/v5/docs/react/reference/useMutation) 用
- `atomWithSuspenseQuery` は [useSuspenseQuery](https://tanstack.com/query/v5/docs/react/reference/useSuspenseQuery) 用
- `atomWithSuspenseInfiniteQuery` は [useSuspenseInfiniteQuery](https://tanstack.com/query/v5/docs/react/reference/useSuspenseInfiniteQuery) 用
- `atomWithMutationState` は [useMutationState](https://tanstack.com/query/v5/docs/react/reference/useMutationState) 用

すべての関数は同じシグネチャに従います。

```ts
const dataAtom = atomWithSomething(getOptions, getQueryClient)
```

最初の `getOptions` パラメータはオブザーバーへの入力を返す関数です。
2番目のオプションの `getQueryClient` パラメータは [QueryClient](https://tanstack.com/query/v5/docs/reference/QueryClient) を返す関数です。

### atomWithQuery の使用方法

`atomWithQuery` は TanStack Query の標準的な [`Query`](https://tanstack.com/query/v5/docs/react/guides/queries) を実装する新しいアトムを作成します。

```jsx
import { atom, useAtom } from 'jotai'
import { atomWithQuery } from 'jotai-tanstack-query'

const idAtom = atom(1)
const userAtom = atomWithQuery((get) => ({
  queryKey: ['users', get(idAtom)],
  queryFn: async ({ queryKey: [, id] }) => {
    const res = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`)
    return res.json()
  },
}))

const UserData = () => {
  const [{ data, isPending, isError }] = useAtom(userAtom)

  if (isPending) return <div>Loading...</div>
  if (isError) return <div>Error</div>

  return <div>{JSON.stringify(data)}</div>
}
```

### atomWithInfiniteQuery の使用方法

`atomWithInfiniteQuery` は `atomWithQuery` に非常に似ていますが、ページネーションされたデータ向けの `InfiniteQuery` 用です。[無限クエリの詳細はこちら](https://tanstack.com/query/v5/docs/framework/react/guides/infinite-queries) を参照してください。

> 既存のデータセットに追加で「もっと読み込む」データを積み上げるリストや「無限スクロール」のレンダリングも非常に一般的な UI パターンです。React Query は、これらのタイプのリストのクエリに useInfiniteQuery という useQuery の便利なバージョンをサポートしています。

標準クエリアトムとの注目すべき違いは、追加の `getNextPageParam` と `getPreviousPageParam` オプションです。これらは追加のページのフェッチ方法をクエリに指示するために使用します。

```jsx
import { atom, useAtom } from 'jotai'
import { atomWithInfiniteQuery } from 'jotai-tanstack-query'

const postsAtom = atomWithInfiniteQuery(() => ({
  queryKey: ['posts'],
  queryFn: async ({ pageParam }) => {
    const res = await fetch(`https://jsonplaceholder.typicode.com/posts?_page=${pageParam}`)
    return res.json()
  },
  getNextPageParam: (lastPage, allPages, lastPageParam) => lastPageParam + 1,
  initialPageParam: 1,
}))

const Posts = () => {
  const [{ data, fetchNextPage, isPending, isError, isFetching }] =
    useAtom(postsAtom)

  if (isPending) return <div>Loading...</div>
  if (isError) return <div>Error</div>

  return (
    <>
      {data.pages.map((page, index) => (
        <div key={index}>
          {page.map((post: any) => (
            <div key={post.id}>{post.title}</div>
          ))}
        </div>
      ))}
      <button onClick={() => fetchNextPage()}>Next</button>
    </>
  )
}
```

### atomWithMutation の使用方法

`atomWithMutation` は TanStack Query の標準的な [`Mutation`](https://tanstack.com/query/v5/docs/framework/react/guides/mutations) を実装する新しいアトムを作成します。

> クエリとは異なり、ミューテーションは通常、データの作成/更新/削除やサーバーサイドエフェクトの実行に使用されます。

```tsx
const postAtom = atomWithMutation(() => ({
  mutationKey: ['posts'],
  mutationFn: async ({ title }: { title: string }) => {
    const res = await fetch(`https://jsonplaceholder.typicode.com/posts`, {
      method: 'POST',
      body: JSON.stringify({
        title,
        body: 'body',
        userId: 1,
      }),
      headers: {
        'Content-type': 'application/json; charset=UTF-8',
      },
    })
    const data = await res.json()
    return data
  },
}))

const Posts = () => {
  const [{ mutate, status }] = useAtom(postAtom)
  return (
    <div>
      <button onClick={() => mutate({ title: 'foo' })}>Click me</button>
      <pre>{JSON.stringify(status, null, 2)}</pre>
    </div>
  )
}
```

### atomWithMutationState の使用方法

`atomWithMutationState` は [`MutationCache`](https://tanstack.com/query/v5/docs/react/reference/useMutationState) 内のすべてのミューテーションへのアクセスを提供する新しいアトムを作成します。

```jsx
const mutationStateAtom = atomWithMutationState((get) => ({
  filters: {
    mutationKey: ['posts'],
  },
}))
```

### プロジェクトで同じ Query Client インスタンスを参照する

プロジェクト内に `useQueryClient()` フックを使用して `QueryClient` オブジェクトを取得してメソッドを呼び出すカスタムフックがある場合、同じ `QueryClient` オブジェクトを参照するために、プロジェクトのルートを `<Provider>` でラップし、`QueryClientProvider` に渡した `queryClient` と同じ値で `queryClientAtom` を初期化してください。

この手順なしでは、`useQueryAtom` は `useQueryClient()` フックを使用してクエリクライアントを取得するフックとは別の `QueryClient` を参照します。

または、`getQueryClient` パラメータで `queryClient` を指定することもできます。

#### 例

以下の例では、ミューテーションフック `useTodoMutation` とクエリ `todosAtom` があります。

ルートの `<App>` ノードに初期化手順を含めています。

両方が同じクエリキー（`'todos'`）を参照しているにもかかわらず、**`Provider` の初期化手順が行われていない場合**、`useTodoMutation` の `onSuccess` invalidation はトリガーされません。

これにより、`todosAtom` は再フェッチが促されないため、古いデータを表示し続けます。

⚠️ 注意：**TypeScript** を使用する場合、`useHydrateAtoms` に queryClient 値を渡す際に Map を使用することを推奨します。動作する例は [レンダリング時の状態初期化ドキュメント](https://jotai.org/docs/guides/initialize-atom-on-render#using-typescript) で確認できます。

```jsx
import { Provider } from 'jotai/react'
import { useHydrateAtoms } from 'jotai/react/utils'
import {
  useMutation,
  useQueryClient,
  QueryClient,
  QueryClientProvider,
} from '@tanstack/react-query'
import { atomWithQuery, queryClientAtom } from 'jotai-tanstack-query'

const queryClient = new QueryClient()

const HydrateAtoms = ({ children }) => {
  useHydrateAtoms([[queryClientAtom, queryClient]])
  return children
}

export const App = () => {
  return (
    <QueryClientProvider client={queryClient}>
      <Provider>
        {/*
   この Provider 初期化手順は、atomWithQuery とアプリの他の部分の両方で
   同じ queryClient を参照するために必要です。これなしでは、
   useQueryClient() フックは別の QueryClient オブジェクトを返します
	*/}
        <HydrateAtoms>
          <App />
        </HydrateAtoms>
      </Provider>
    </QueryClientProvider>
  )
}

export const todosAtom = atomWithQuery((get) => {
  return {
    queryKey: ['todos'],
    queryFn: () => fetch('/todos'),
  }
})

export const useTodoMutation = () => {
  const queryClient = useQueryClient()

  return useMutation(
    async (body: todo) => {
      await fetch('/todo', { Method: 'POST', Body: body })
    },
    {
      onSuccess: () => {
        void queryClient.invalidateQueries(['todos'])
      },
      onError,
    }
  )
}
```

### SSR サポート

すべてのアトムは Next.js アプリや Gatsby アプリなどのサーバーサイドレンダリングアプリのコンテキスト内で使用できます。SSR アプリ内で React Query がサポートする [両方のオプション](https://tanstack.com/query/v5/docs/framework/react/guides/ssr) を使用できます：[ハイドレーション](https://tanstack.com/query/v5/docs/react/guides/ssr#using-the-hydration-apis) または [`initialData`](https://tanstack.com/query/v5/docs/react/guides/ssr#get-started-fast-with-initialdata)。

### エラーハンドリング

フェッチエラーはスローされ、ErrorBoundary でキャッチできます。
再フェッチすることで一時的なエラーから回復できる場合があります。

詳細は [動作する例](https://codesandbox.io/s/4gfp6z) を参照してください。

### Devtools

Devtools を使用するには、追加でインストールする必要があります。

```bash
npm install @tanstack/react-query-devtools
```

`<QueryClientProvider />` 内に `<ReactQueryDevtools />` を配置するだけです。

```tsx
import {
  QueryClientProvider,
  QueryClient,
  QueryCache,
} from '@tanstack/react-query'
import { ReactQueryDevtools } from '@tanstack/react-query-devtools'
import { queryClientAtom } from 'jotai-tanstack-query'

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: Infinity,
    },
  },
})

const HydrateAtoms = ({ children }) => {
  useHydrateAtoms([[queryClientAtom, queryClient]])
  return children
}

export const App = () => {
  return (
    <QueryClientProvider client={queryClient}>
      <Provider>
        <HydrateAtoms>
          <App />
        </HydrateAtoms>
      </Provider>
      <ReactQueryDevtools />
    </QueryClientProvider>
  )
}
```

## v0.8.0 への移行

### アトムシグネチャの変更

すべてのアトムシグネチャが TanStack Query との一貫性を高めるために変更されました。
v0.8.0 ではタプルのアトムではなく単一のアトムのみを返すため、`atomsWithSomething` から `atomWithSomething` に名前が変更されました。

```diff

- const [dataAtom, statusAtom] = atomsWithSomething(getOptions, getQueryClient)
+ const dataAtom = atomWithSomething(getOptions, getQueryClient)

```

### 簡略化された返却構造

`jotai-tanstack-query` の以前のバージョンでは、クエリアトム `atomsWithQuery` と `atomsWithInfiniteQuery` はアトムのタプル `[dataAtom, statusAtom]` を返していました。この設計はデータとそのステータスを2つの異なるアトムに分離していました。

#### atomWithQuery と atomWithInfiniteQuery

- `dataAtom` は実際のデータ（`TData`）へのアクセスに使用されていました。
- `statusAtom` は `isPending`、`isError` などの追加属性を含むステータスオブジェクト（`QueryObserverResult<TData, TError>`）を提供していました。

v0.8.0 では、`atomWithQuery` と `atomWithInfiniteQuery` に置き換えられ、単一の `dataAtom` のみを返すようになりました。この `dataAtom` は `QueryObserverResult<TData, TError>` を直接提供し、Tanstack Query のバインディングの動作に近づいています。

新しいバージョンに移行するには、別々の `dataAtom` と `statusAtom` の使用を、データとステータス情報の両方を含む統合された `dataAtom` に置き換えてください。

```diff
- const [dataAtom, statusAtom] = atomsWithQuery(/* ... */);
- const [data] = useAtom(dataAtom);
- const [status] = useAtom(statusAtom);

+ const dataAtom = atomWithQuery(/* ... */);
+ const [{ data, isPending, isError }] = useAtom(dataAtom);
```

#### atomWithMutation

`atomsWithQuery` と `atomsWithInfiniteQuery` と同様に、`atomWithMutation` もタプルのアトムではなく単一のアトムを返します。アトム値の返却型は `MutationObserverResult<TData, TError, TVariables, TContext>` です。

```diff

- const [, postAtom] = atomsWithMutation(/* ... */);
- const [post, mutate] = useAtom(postAtom); // post からミューテーションステータスにアクセス; mutate() でミューテーションを実行

+ const postAtom = atomWithMutation(/* ... */);
+ const [{ data, error, mutate }] = useAtom(postAtom); // 同じアトムからミューテーション結果と mutate メソッドにアクセス

```
