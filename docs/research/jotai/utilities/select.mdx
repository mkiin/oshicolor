## selectAtom

⚠️ 名前とは異なり、`selectAtom` はエスケープハッチとして提供されています。使用することは100%純粋なアトムモデルを構築しないことを意味します。派生アトムの使用を優先し、`equalityFn` または `prevSlice` が不可避な場合のみ `selectAtom` を使用してください。

### シグネチャ

```ts
function selectAtom<Value, Slice>(
  anAtom: Atom<Value>,
  selector: (v: Value, prevSlice?: Slice) => Slice,
  equalityFn: (a: Slice, b: Slice) => boolean = Object.is,
): Atom<Slice>;
```

この関数は、`selector` によって決定される元のアトムの値の関数として値を持つ派生アトムを作成します。
セレクター関数は元のアトムが変化するたびに実行され、`equalityFn` が派生値が変化したと報告した場合にのみ派生アトムを更新します。
デフォルトでは、`equalityFn` は参照等価性ですが、必要に応じて派生値を安定化するためにお気に入りの深い等価比較関数を指定できます。

### 例

```js
const defaultPerson = {
  name: {
    first: "Jane",
    last: "Doe",
  },
  birth: {
    year: 2000,
    month: "Jan",
    day: 1,
    time: {
      hour: 1,
      minute: 1,
    },
  },
};

// 元のアトム
const personAtom = atom(defaultPerson);

// person.name を追跡します。name.first も name.last も実際には変わらない場合でも、
// person.name オブジェクトが変化したときに更新されます。
const nameAtom = selectAtom(personAtom, (person) => person.name);

// person.birth を追跡します。year、month、day、hour、または minute が変化したときに更新されます。
// deepEquals を使用することで、birth フィールドが同じデータを含む新しいオブジェクトに
// 置き換えられた場合（例: データベースから person を再読み込みした場合）、
// このアトムは更新されません。
const birthAtom = selectAtom(personAtom, (person) => person.birth, deepEquals);
```

### 安定した参照を保持する

いつものように、render サイクルで `useAtom` を使用する際に無限ループを防ぐために、
`useAtom` に安定したアトムの参照を提供する必要があります。
`selectAtom` では、ベースアトムとセレクターの**両方**が安定している必要があります。

```js
const [value] = useAtom(selectAtom(atom(0), (val) => val)); // これは無限ループを引き起こします
```

これらの制約を満たすための複数のオプションがあります：

```js
const baseAtom = atom(0); // 安定
const baseSelector = (v) => v; // 安定
const Component = () => {
  // 解決策1：「useMemo」で結果のアトム全体をメモ化
  const [value] = useAtom(useMemo(() => selectAtom(baseAtom, (v) => v), []));

  // 解決策2：「useCallback」でインラインコールバックをメモ化
  const [value] = useAtom(
    selectAtom(
      baseAtom,
      useCallback((v) => v, []),
    ),
  );

  // 解決策3：すべての制約がすでに満たされている
  const [value] = useAtom(selectAtom(baseAtom, baseSelector));
};
```
