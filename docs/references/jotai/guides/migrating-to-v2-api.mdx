RFC: https://github.com/pmndrs/jotai/discussions/1514

Jotai v1 は 2022年6月にリリースされ、様々なフィードバックが寄せられました。
React もプロミスへのファーストクラスサポートを提案しています。
Jotai v2 は新しい API を持ちます。

残念ながら、新機能とともにいくつかの破壊的変更があります。

### 新機能

#### バニラライブラリ

Jotai はバニラ（非 React）関数と React 関数を別々に提供します。
`jotai/vanilla` などの代替エントリーポイントから提供されます。

#### Store API

Jotai はストアインターフェースを公開しており、アトムの値を直接操作できます。

```js
import { createStore } from "jotai"; // または 'jotai/vanilla' から

const store = createStore();
store.set(fooAtom, "foo");

console.log(store.get(fooAtom)); // "foo" を出力

const unsub = store.sub(fooAtom, () => {
    console.log("fooAtom value in store is changed");
});
// サブスクライブを解除するには unsub() を呼び出す
```

ストアを渡すために独自の React Context を作成することもできます。

#### より柔軟なアトムの `write` 関数

write 関数は複数の引数を受け取り、値を返すことができます。

```js
atom(
  (get) => get(...),
  (get, set, arg1, arg2, ...) => {
    ...
    return someValue
  }
)
```

### 破壊的変更

#### 非同期アトムはもはや特別ではない

非同期アトムは Promise 値を持つ通常のアトムに過ぎません。
アトムのゲッター関数はプロミスを解決しません。
一方、`useAtom` フックは引き続きプロミスを解決します。

`splitAtom` などのユーティリティは同期アトムを期待しており、
非同期アトムでは機能しません。

#### WritableAtom の型が変更された（TypeScript のみ）

```ts
// 旧
WritableAtom<Value, Arg, Result extends void | Promise<void>>

// 新
WritableAtom<Value, Args extends unknown[], Result>
```

一般的に、`WritableAtom` 型は直接使用しないようにしてください。

#### 一部の関数が削除された

- Provider の `initialValues` prop が削除されました。`store` の方が柔軟なためです。
- Provider の scope prop が削除されました。独自のコンテキストを作成できるためです。
- `abortableAtom` ユーティリティが削除されました。この機能がデフォルトで含まれるためです。
- `waitForAll` ユーティリティが削除されました。`Promise.all` がそのまま機能するためです。

### 移行ガイド

#### 非同期アトム

非同期アトムの read 関数の `get` 関数はプロミスを解決しないため、
`await` または `.then()` を付ける必要があります。

要するに、変更は次のようになります。
（TypeScript ユーザーの場合、型がどこを変更すべきか教えてくれます。）

##### 旧 API

```js
const asyncAtom = atom(async () => "hello");
const derivedAtom = atom((get) => get(asyncAtom).toUppercase());
```

##### 新 API

```js
const asyncAtom = atom(async () => "hello");
const derivedAtom = atom(async (get) => (await get(asyncAtom)).toUppercase());
// または
const derivedAtom = atom((get) => get(asyncAtom).then((x) => x.toUppercase()));
```

#### Provider の `initialValues` prop

##### 旧 API

```jsx
const countAtom = atom(0)

  // コンポーネント内で
  <Provider initialValues={[[countAtom, 1]]}>
    ...
```

##### 新 API

```jsx
const countAtom = atom(0)

const HydrateAtoms = ({ initialValues, children }) => {
  useHydrateAtoms(initialValues)
  return children
}

  // コンポーネント内で
  <Provider>
    <HydrateAtoms initialValues={[[countAtom, 1]]}>
      ...
```

#### Provider の `scope` prop

##### 旧 API

```jsx
const myScope = Symbol()

  // 親コンポーネント
  <Provider scope={myScope}>
    ...
  </Provider>

  // 子コンポーネント
  useAtom(..., myScope)
```

##### 新 API

```jsx
const MyContext = createContext()
const store = createStore()

  // 親コンポーネント
  <MyContext.Provider value={store}>
    ...
  </MyContext.Provider>

  // 子コンポーネント
  const store = useContext(MyContext)
  useAtom(..., { store })
```

#### `abortableAtom` ユーティリティ

以前の `abortableAtom` ユーティリティは不要になりました。
通常の `atom` でサポートされるようになったためです。

##### 旧 API

```js
const asyncAtom = abortableAtom(async (get, { signal }) => {
 ...
}
```

##### 新 API

```js
const asyncAtom = atom(async (get, { signal }) => {
  ...
}
```

#### `waitForAll` ユーティリティ

以前の `waitForAll` ユーティリティは不要になりました。
ネイティブの Promise API が使用できるためです。

##### 旧 API

```js
const allAtom = waitForAll([fooAtom, barAtom]);
```

##### 新 API

```js
const allAtom = atom((get) => Promise.all([get(fooAtom), get(barAtom)]));
```

render 関数でアトムを作成すると[無限ループ](../core/atom.mdx#note-about-creating-an-atom-in-render-function)が発生する可能性があることに注意してください。

#### 非同期アトムと `splitAtom` ユーティリティ（または他のユーティリティ）

`splitAtom` ユーティリティは同期アトムのみを受け入れます。
渡す前に非同期アトムをアンラップする必要があります。

これは `jotai-tanstack-query` の `atomsWithQuery` などの他のユーティリティにも適用されます。

##### 旧 API

```js
const splittedAtom = splitAtom(asyncArrayAtom);
```

##### 新 API

```js
const splittedAtom = splitAtom(unwrap(asyncArrayAtom, () => []));
```

執筆時点では、`unwrap` は不安定でドキュメント化されていません。
代わりに、ローディング状態をより詳細に制御できる `loadable` を使用できます。
`<Suspense>` を使用する必要がある場合は、atoms-in-atom パターンが役立ちます。

詳細については、以下のディスカッションを参照してください：

- https://github.com/pmndrs/jotai/discussions/1615
- https://github.com/jotaijs/jotai-tanstack-query/issues/21
- https://github.com/pmndrs/jotai/discussions/1751

### その他の変更

#### ユーティリティ

- `atomWithStorage` ユーティリティの `delayInit` がデフォルトとして削除されました。また、初回レンダリングでは常に `initialValue` をレンダリングし、以降のレンダリングでは（存在する場合）格納された値をレンダリングします。新しい動作は v1 とは異なります。詳細は https://github.com/pmndrs/jotai/discussions/1737 を参照してください。
- `useHydrateAtoms` は書き込み可能なアトムのみを受け入れるようになりました。

#### import 文

v2 API はライブラリ作者と非 React ユーザー向けに代替エントリーポイントからも提供されます。

- `jotai/vanilla`
- `jotai/vanilla/utils`
- `jotai/react`
- `jotai/react/utils`

```js
// v1.11.0 から利用可能
import { atom } from "jotai/vanilla";
import { useAtom } from "jotai/react";

// v2.0.0 から利用可能
import { atom } from "jotai"; // 'jotai/vanilla' と同じ
import { useAtom } from "jotai"; // 'jotai/react' と同じ
```

注意：ESM を使用していない場合は、より良いツリーシェイキングのために `jotai` の代わりに `jotai/vanilla` などを使用することを推奨します。
